1 Установка RabbitMQ для SLES12 SP3

1.1 Общая информация

Документ содержит руководство администратора по установке, настройке и запуску сервера брокера сообщений RabbitMQ на SLES 12 SP3.

1.2 Подготовка

Примечание. Все требуемые (перечисленные ниже) пакеты для установки RabbitMQ также можно получить, сделав запрос в службу техподдержки ООО «ЭМА»: support@ema.ru.

1.2.1 RabbitMQ

Рекомендуется ставить версию 3.7.10. Данная версия RabbitMQ для SLES (RPM-пакет rabbitmq-server-3.7.10-1.suse.noarch.rpm) доступна по ссылке:

https://github.com/rabbitmq/rabbitmq-server/releases/download/v3.7.10/rabbitmq-server-3.7.10-1.suse.noarch.rpm.

1.2.2 Socat

Для установки RabbitMQ Server требуется утилита Socat. Устанавливается с DVD дистрибутива SLES12 (в дистрибутиве SLES12 SP3 содержится версия socat - 1.7.2.4-3.1).

1.2.3 UnixODBC

Пакет erlang требует установленного UnixODBC. Устанавливается с DVD дистрибутива SLES12 (в дистрибутиве SLES12 SP3 содержится версия unixODBC 2.3.4-6.5).

1.2.4 Erlang

Для работы RabbitMQ версии 3.7.10 требуется Erlang версии не ниже 19.3

Примечание. У Erlang версии 19 поддержка версий RabbitMQ 3.6.4 – 3.7.10.

Для RabbitMQ версии 3.7.10 рекомендуется использовать Erlang версии 19.3.2-8.1.

Для работы Erlang также требуется установка пакета erlang-epmd (Erlang Port Mapper daemon). Расположение аналогично (пакет erlang-epmd-19.3.2-8.1).

 

1.3 Установка 

Дополнительную информацию по установке RabbitMQ можно прочитать в инструкции на официальном сайте: http://www.rabbitmq.com/install-rpm.html.

Примечание. Все действия должны выполняться с правами пользователя root.

1.3.1 Установка программных пакетов

Скопировать на сервер пакеты erlang-19.3.2-8.1.x86_64.rpm, erlang-epmd-19.3.2-8.1.x86_64.rpm и rabbitmq-server-3.7.10-1.suse.noarch.

Установить erlang вместе с erlang-empd:

# rpm -ivh erlang-epmd-19.3.2-8.1.x86_64.rpm erlang-19.3.2-8.1.x86_64.rpm

warning: erlang-epmd-19.3.2-8.1.x86_64.rpm: Header V3 RSA/SHA1 Signature, key ID 1e639224: NOKEY

Preparing...                          ################################# [100%]

Updating / installing...

   1:erlang-19.3.2-8.1                ################################# [ 50%]

   2:erlang-epmd-19.3.2-8.1           ################################# [100%]

Updating /etc/sysconfig/erlang...

Установить RabbitMQ:

# rpm -i rabbitmq-server-3.7.10-1.suse.noarch.rpm

warning: rabbitmq-server-3.7.10-1.suse.noarch.rpm: Header V4 RSA/SHA512 Signature, key ID 6026dfca: NOKEY

Проверить работу сервера:

# systemctl stop rabbitmq-server

 

# systemctl start rabbitmq-server

 

# systemctl status rabbitmq-server

1.3.3 Создание пользователя admin для управления через WEB-консоль

При установке в RabbitMQ создаётся единственный пользователь guest. Пользователю guest назначен тег (Tag) administrator, но доступа к WEB-интерфейсу у него нет. Поэтому необходимо создать пользователя (отличного от guest) с правами администратора и возможностью использования WEB-интерфейса, например, пользователя admin.

Создать пользователя admin:

# rabbitmqctl add_user admin passme

Creating user "admin" ...

 

# rabbitmqctl list_users
Listing users ...
admin   []
guest   [administrator]

Добавить пользователя admin в группу:

# rabbitmqctl set_user_tags admin administrator
Setting tags for user "admin" to [administrator] ...

 

# rabbitmqctl list_users
Listing users ...
admin   [administrator]
guest   [administrator]

Назначить права пользователю admin:

# rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"
Setting permissions for user "admin" in vhost "/" ...

Теперь пользователь admin может использоваться для входа в WEB-интерфейс управления (Рисунок 1).

1.4 Настройка

В данном разделе описываются общие вопросы по настройке брокера сообщений RabbitMQ.

Информация о дальнейшей конфигурации для РСДУ содержится в «Настройка RabbitMQ для функционирования сервисов РСДУ. Руководство администратора».

 

1.4.1 Файл конфигурации RabbitMQ

Для тонкой настройки работы брокера сообщений RabbitMQ используется файл конфигурации rabbitmq.config. В нем содержатся настройки работы сервера и плагинов. На текущий момент данные настройки для работы сервисов РСДУ не используются. По умолчанию файл rabbitmq.config отсутствует, поэтому на вкладке «Overview» WEB-консоли RabbitMQ можно увидеть сообщение (not found), а также путь, откуда сервер пытается его прочитать (Рисунок 2).

Рисунок 2 – Путь до файла конфигурации RabbitMQ

Если требуется внести коррективы, необходимо скачать пример файла конфигурации с сайта по ссылке, указанной на странице: http://www.rabbitmq.com/configure.html#configuration-file, и поместить в директорию, указанную в WEB-консоли RabbitMQ. Следует убедиться в том, что этот файл доступен для чтения.

Чтобы изменения применились, требуется перезагрузить сервер:

# service rabbitmq-server stop

 

# service rabbitmq-server start

2 Администрирование RabbitMQ

2.1 Запуск и останов сервера RabbitMQ

Для запуска сервера необходимо выполнить следующую команду:

# service rabbitmq-server start

Для остановки:

# service rabbitmq-server stop

2.2 Контроль лог-файлов

Лог-файл брокера сообщений располагается в каталоге /var/log/rabbitmq

Основной лог работы брокера сообщений находится в файле, имя которого состоит из: rabbit@<hostname>.log

Например: rabbit@rmq1.log

Пример сообщения о запуске брокера сообщений в лог-файле:

…

=INFO REPORT==== 8-Nov-2017::16:46:24 ===

Server startup complete; 6 plugins started.

 * rabbitmq_management

 * rabbitmq_management_agent

 * rabbitmq_web_dispatch

…

 

 

Приложение А. Используемые порты

В документации к RabbitMQ перечислены следующие задействованные номера сетевых портов:

ü  4369 (epmd), 25672 (Erlang distribution);

ü  5672, 5671 (AMQP 0-9-1 without and with TLS);

ü  15672 (if management plugin is enabled);

ü  61613, 61614 (if STOMP is enabled);

ü  1883, 8883 (if MQTT is enabled).
