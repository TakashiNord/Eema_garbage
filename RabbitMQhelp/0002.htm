1 Введение

1.1 Общая информация

Документ содержит описание по настройке брокера сообщений RabbitMQ для функционирования сервисов РСДУ.

1.2 Необходимые условия

ü  должен быть установлен и настроен брокер сообщений RabbitMQ;

ü  должен быть организован доступ к WEB-интерфейсу консоли управления (адрес, логин и пароль администратора RabbitMQ);

ü  должен быть в наличии файл предопределённой конфигурации для РСДУ, который содержит описание пользователей и точек обмена для сервисов РСДУ (актуальная версия файла конфигурации RabbitMQ входит в состав дистрибутива РСДУ, начиная с версии 5.21.3.0).

1.3 Необходимые знания

Пользователю, выполняющему настройку брокера сообщений RabbitMQ, необходимо обладать навыками работы с браузером.

 

2 Настройка

2.1 Создание виртуального хоста

По умолчанию на сервере создан один виртуальный хост «/». При необходимости можно создать отдельный виртуальный хост, для которого будет применяться последующая конфигурация RabbitMQ. Таких хостов может быть несколько.

Создание виртуального хоста (VHOST) осуществляется через WEB-консоль RabbitMQ. Следует открыть вкладку «Admin», выбрать пункт «Virtual Hosts», ввести в секции «Add a new virtual host» в поле «Name» название, например, «rsdu» и подтвердить создание с помощь кнопки «Add virtual host» (Рисунок 1).

Рисунок 1 – Создание виртуального хоста

 

2.2 Загрузка файла конфигурации РСДУ

Необходимо выполнить загрузку предопределенной конфигурации для РСДУ, которая содержит определения (описания) пользователей и точек обмена для сервисов РСДУ.

Отдельным архивом RSDU5_RMQ-CONF-<version>.tar.gzip распространяются файлы:

ü  rsdu_rmq_config_full.json – содержит полный набор определений;

ü  rsdu_rmq_config_users.json – содержит определения пользователей;

ü  rsdu_rmq_config_exchanges.json – содержит определения точек обмена;

ü  rsdu_rmq_config_queues.json – содержит определения очередей.

Файлы, содержащие часть определений, используются для восстановления настроек брокера.

Для загрузки файла необходимо открыть вкладку «Overview» WEB-консоли RabbitMQ.

Внизу страницы, в секции «Import / export definitions» в части «Import», в поле «Definitions file», нажав на кнопку «Обзор…», выбрать файл конфигурации (например, rsdu_rmq_config.json).

Указать виртуальный хост «Virtual host», к которому следует применить конфигурацию, например, «rsdu», если создавался отдельный виртуальный хост.

Далее следует подтвердить выбор с помощью кнопки «Upload broker definitions».

Примечание 1. Из файла импортируются только те определения, которые не описаны в настройках брокера. Существующие определения не переписываются.

Примечание 2. Версия конфигурационного файла должна совпадать с версией установленного брокера сообщений RabbitMQ.

2.3 Настройка пользователей

После импорта конфигурации РСДУ на вкладке «Admin» в списке «All users» появится пользователь rsdu.

Необходимо убедиться в наличии прав на созданный ранее виртуальный хост.

В колонке «Can access virtual hosts» для пользователей rsdu и admin в списке должно присутствовать имя виртуального хоста, например «rsdu» или «/», если отдельный виртуальный хост не создавался.

Рисунок 3 – Проверка прав на созданный виртуальный хост

Если хоста нет в списке, необходимо его добавить, нажав левой кнопкой мыши на имя пользователя в таблице. Откроется раздел установки прав «Set permission». Выбрать в списке «Virtual Host» rsdu и нажать кнопку «Set permission» (Рисунок 4).

Рисунок 4 – Назначение прав на созданный виртуальный хост

2.4 Проверка конфигурации

После загрузки предопределенной конфигурации должны появиться точки обмена, необходимые для работы сервисов РСДУ.

Для проверки необходимо перейти на вкладку «Exchanges» и выбрать виртуальный хост, к которому применялась конфигурация, в списке «Virtual host» вверху справа. В нашем случае – rsdu. Если отдельный виртуальный хост не создавался, то интерфейса выбора не будет (так как хост один, по умолчанию – «/»). В списке должны отобразиться точки обмена РСДУ (rsdu.events.disptag, rsdu.signal и другие, см. Рисунок 5).
