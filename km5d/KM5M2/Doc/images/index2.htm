1. Введение

   Программа "km_5m_.exe" входит в комплект сервисного программного обеспечения (ПО) и работает в опереционных системах Windows 95/98/Me/XP/NT/2000/2003 и др. с установленным пакетом MSOffice2000-2003 (с Access). Программа может использоваться как в составе данного комплекта ПО, так и запускаться отдельно и выполняет следующие функции:

  • считывания на ПК почасовых, посуточных, помесячных, погодовых архивов и архивов событий с теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) и с устройств переноса данных УПД, АП-5-3;
  • сохранения архивных данных в файлах на персональном компьютере, дополнением новых записей к считанным ранее архивам для дальнейшего просмотра и распечатки отчётов программой "km5m_aXXXX_X_XX.exe" из комплекта сервисного программного обеспечения;
  • мониторинга мгновенных параметров тепло/водопотребления для теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) и диаграмм в режиме реального времени;
  • дистанционного управления теплосчётчиками/расходомерами КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3);
  • настройки и считывания конфигурации теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) (на этапе отладки!!!).
  Считывание архивных данных из теплосчётчиков/расходомеров описывается в разделе "Загрузка архивных данных из теплосчётчиков/расходомеров в базу данных ПК"


  Способы подключения к приборам описаны в руководствах по эксплуатации:

  • "Теплосчётчик КМ-5М. Руководство по эксплуатации".
  • "Теплосчётчик КМ5-Б-3. Руководство по эксплуатации".
  • "Расходомер РМ5-Б-3. Руководство по эксплуатации".
  • "Периферийные устройства. Руководство по монтажу и эксплуатации".

  Сохранение, обработка и распечатка ведомостей параметров теплопотребления

  Сохранение, обработка и распечатка ведомостей параметров тепло/водопотребления производится программой "km5m_aXXXX_X_XX.mdb" из комплекта сервисного программного обеспечения. Получить доступ к тексту можно через пункт "Руководство пльзователя" из меню "Справка" описываемой программы.

2. Установка и настройка программы

  Программа считывания базы данных, мониторинга параметров тепло/водопотребления для теплосчётчиков/расходомеров запускается как отдельно, так и из программы распечаток ведомостей "km5m_aXXXX_X_XX.mdb". При установке ПО рекомендуется скопировать в рабочую директорию сервисное программное обеспечение:

  1. программа "km_5m_.exe" - считывание базы данных, мониторинг параметров тепло/водопотребления;
  2. программа "km5m_aXXXX_X_XX.mdb" - работа со считанными архивами;
  3. программа "km5mpult.exe" - дистанционный видеоконтроль и мониторинг вычислителей КМ-5М, КМ-5-Б3(Б1), РМ-5-Б3(Б1);
  4. программа "AVRPC.exe" - технологическая DOS-программа;
  5. программа "HWIEW.exe";
  6. подкаталог с содержимым "KM5M_HLP".
  7. подкаталог с содержимым "Doc".

  При первом запуске программы необходимо выполнить настройку в окне "Настройка параметров", открываемом через пункт "Параметры" в меню "Настройка". Появится окно (Рис.1).

Рисунок 1. Окно "Настройка параметров".

  Следует выполнить настройки:

  1. Отметить используемый COM-порт или несколько COM-портов, остальные "отключить". Возможен выбор виртуальных COM-портов, нарпример, при подключении через интеграторы сети или автоматический преобразователь интерфейса АПИ-USB/RS485,КСПД-5.
  2. Если установлен флаг "Автоматический поиск прибора при запуске", то программа начинает поиск подключенных приборов сразу после запуска. Данный режим удобно использовать при подключении одного теплосчётчика/расходомера. Когда подключены два и более прибора, перед поиском необходимо внести номера приборов (Разд.3.1.). Рекомендуется отключить автоматический поиск.
  3. Флаг "Подбор скорости при неудачном соединении" рекомендуется оставить включённым, так как теплосчётчик/расходомер может быть настроен на скорость 19200 бод при считывании архивов БД. При установленном данном флаге программа "распознаёт" эту скорость. Иначе теплосчётчик/расходомер не подключится, и потребуется его перезапустить, что не всегда представляется возможным. При считывании с УПД (АП-5-3) можно сбросить этот флаг, что ускорит процесс поиска прибора.
  4. Параметр "Время ожидания ответа" можно оставить без изменения, а редактировать в дальнейшем при необходимости. Например, при подключении к приборам по длинным линиям связи (RS-485) возможно потребуется увеличить этот параметр до 1000.
  5. Параметр "Дискретность мониторинга" можно оставить без изменения. Он используется при мониторинге параметров тепло/водопотребления. Величину этого параметра можно изменять непосредственно перед запуском мониторинга в соответствии с рекомендациями, приведёнными в разделе: Мониторинг параметров тепло/водопотребления для теплосчётчиков/расходомеров
  6. Сохранить настройки можно, нажав кнопку "Запись". При этом закроется окно "Настройка параметров". Если нажать кнопку "Отмена", то окно закроется без сохранения параметров.
Параметры настройки записываются в файл "KM_5M_.INI" в рабочей директории. Если удалить этот файл, то параметры будут потеряны, и при ближайшей запуске программы сформируется новый аналогичный файл с исходными параметрами по умолчанию, которые далее можно перенастроить.

3. Загрузка архивных данных из теплосчётчиков/расходомеров в базу данных ПК

   Загрузка баз данных из теплосчётчиков/расходомеров может осуществляться следующим образом:

  • при подключении одного или нескольких теплосчётчиков/расходомеров (объединенных сетью RS-485) к последовательному порту ПК через адаптер периферии АП-5-2/3 или преобразователь интерфейса RS485/RS232 АПИ-5; АПИ-USB/RS485
  • по телефонной сети через модем;
  • по локальной сети и Internet через интегратор сети с разъёмом Ethernet, КСПД-5;
  • посредством устройств переноса данных УПД-8/16/32, АП-5-3;
   Далее приведено более подробное описание загрузки архивных данных по каждому из описанных выше способов.

3.1. Загрузка баз данных по сети RS-485

   Линия "RS-485" позволяет загружать архивные данные из одного или нескольких теплосётчиков/расходомеров (объединённых сетью RS-485), подключённых к последовательному порту компьютера через АП-5-2/3, АПИ-5 и нуль-модемный кабель или через автоматический преобразователь интерфейса АПИ-USB/RS485 без нуль-модемного кабеля.

Последовательность действий:

   

1. Запустить программу "km_5m_.exe". Откроется основное окно, приведённое на рисунке 2. При установленном флаге "Автоматический поиск прибора при запуске" см. раздел "Установка и настройка программы" запустится процесс поиска подключённого теплосчётчика/расходомера. При успешном завершении поиска, в основном окне отобразится информация о приборе (Рисунок 2), после чего можно сразу переходить к пункту 4 данного раздела. При неуспешном завершении выдаётся сообщение "Устройство не обнаружено!". Тогда необходимо проверить исправность соединения и перейти к пункту 2 данного раздела.

Рисунок 2 Основное окно программы.

   2. Если подключен один теплосчётчик/расходомер, запустить процесс поиска теплочётчика/расходомера одним из способов:

  • активизировать поле ввода "№" (Рисунок 2, поз.3) и нажать клавишу <Ввод>;
  • в меню "Работа с прибором" запустить команду "Поиск прибора";
  • нажать значок "Найти прибор" (Рисунок 2, поз.1).
   На экран выводится текущая информация об состоянии поиска. Процесс можно прервать в любой момент с помощью кнопки "Прервать".
   При успешном завершении поиска в основном окне отобразится информация о приборе (Рисунок 4). Иначе выдаётся сообщение "Устройство не обнаружено!". Тогда необходимо проверить исправность соединения и повторить действия, начиная с пункта 2. При многократных неудачных попытах рекомендуется перезапустить прибор и перейти к пункту 2 данного раздела.     При считывании группы теплосчётчиков/расходомеров ввести заводские номера теплосчётчиков/расходомеров в поле ввода "№" (Рис.2, поз.3), либо выбрать из предварительно созданного списка, нажав стрелку вниз (Рис.2, поз.3). При вводе номера нули в старших разрядах можно опустить. Список номеров приборов в сети создаётся в окне "Приборы в сети" (Рис.3), вызываемом нажатием кнопки "Приборы в сети" , либо двойным щелчком левой кнопки мыши на поле ввода "№" (Рис.2,поз.3).


Рисунок 3. Окно "Приборы в сети".

   В данном окне слева в списке отображаются уже введённые номера приборов. Чтобы добавить новый номер, его необходимо ввести в поле "Введите адрес:", после чего нажать на кнопку "<< Добавить". При вводе номера нули в старших разрядах можно опустить. Удалить номер из списка можно, выделив его в списке и нажать на кнопке "<< Удалить".
   Результаты редактирования списка можно сохранить, нажав на кнопке "Запись", либо отказаться от сохранения, нажав на кнопке "Отмена". В обоих случаях закроется окно "Приборы в сети".


Рисунок 4. Вид основного окна после установки связи с прибором.

   4. В поле "Скорость" (Рисунок 4, поз.2) отображается скорость обмена с подключённым прибором. Теплосчётчик/расходомер может быть настроен на скорость обмена 9600/19200 бод, причем, настройка сохраняется до ближайшего перезапуска прибора. После перезапуска скорость устанавливается 9600 бод.
   Для увеличения быстродействия целесообразно перейти на скорость 19200 бод. Но при этом прибор должен быть подключен к компьютеру через преобразователь интерфейса АПИ-5. Для этого необходимо в поле "Скорость" нажать значок "стрелка вниз" и в списке выбрать "19200". Запустится процесс перевода прибора на скорость 19200 бод установки связи с прибором. Если будет выдано сообщение "Устройство не обнаружено!", то повторить попытку, начиная с пункта 2 данного раздела. При многократных неудачных попытах рекомендуется ерезапустить программу или перезапустить прибор и перейти к пункту 2 данного раздела.
   Аналогично можно перейти на скорость 9600 бод, если наблюдаются сбои по связи на скорости 19200 бод.


   5. Открыть окно "Архив" (Рисунок 5) одним из способов:
  • в меню "Работа с прибором" запустить команду "База данных";
  • нажать значок "Найти прибор" (Рисунок 4, поз.1).

Рисунок 5. Окно "Архив" на этапе считывания заголовка архива.

   При открытии окна автоматически начинается считывание заголовка архива. Процесс считывания отображается на индикации (Рисунок 5, поз.1). На этом этапе можно прервать дальнейшую работу с архивом, нажав на кнопку "Прервать" (Рисунок 5, поз.2), при этом окно "Архив" закроется.
   При успешном завершении считывания заголовков архива окно принимает вид, приведённый на рисунке 6. Можно завершить работу с архивом, нажав кнопку "Выход" (Рисунок 6, поз.2), либо продолжить.


Рисунок 6. Окно "Архив" после считывания заголовка архива.

   6. Нажать кнопку "Считать в файл" (Рисунок 6, поз.1). Если на ПК уже имеется считанный ранее файл архива данного прибора, то перед запуском процесса появляется предупреждение (Рис.7), и предлагаются на выбор два варианта:

  • Дополнить файл архива новыми записями (кнопка "Да");
  • Создать новый архив (кнопка "Нет").

Рисунок 7. Предупреждение о наличии файла архива.

   При выборе первого варианта не тратится время на повторное считывание уже имеющихся записей архива. Второй вариант целесообразно выбрать, если есть сомнения в корректности данных при предыдущем считывании архива.

   7. При считывании данных архива в файл в окне "Архив" экран принимает вид, приведённый на рисунке 8.
