1. Введение

   Сервисное программное обеспечение работает в операционных системах Windows: 95/98/Me/XP/NT/2000/2003/2007 и др. и выполняет следующие функции:

  • считывания на ПК почасовых, посуточных, помесячных, погодовых архивов и архивов событий с теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) и с устройств переноса данных УПД-8/16/32, АП-5-3;
  • сохранения архивных данных в файлах на персональном компьютере, дополнением новых записей к считанным ранее архивам для дальнейшего просмотра и распечатки отчётов;
  • формирование, просмотр и распечатка почасовых, посменных, посуточных ведомостей параметров тепло/водопотребления;
  • графическое представление параметров тепло/водопотребления;
  • формирование, просмотр и распечатка архива событий/ошибок;
  • экспорт в файл Excel почасовых, посменных, посуточных архивных данных за заданный период времени;
  • экспорт в текстовый файл данных событий/ошибок за заданный период.
  • мониторинга мгновенных параметров тепло/водопотребления для теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) и диаграмм в режиме реального времени;
  • дистанционного управления теплосчётчиками/расходомерами КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3);
  • настройки и считывания конфигурации теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) .

   Монтаж и подключение к приборам описаны в руководствах по эксплуатации:

  • "Теплосчётчик КМ-5М. Руководство по эксплуатации".
  • "Теплосчётчик КМ5-Б-3. Руководство по эксплуатации".
  • "Расходомер РМ5-Б-3. Руководство по эксплуатации".
  • "Периферийные устройства. Руководство по монтажу и эксплуатации".

   Сервисная программа используется для сервисного обслуживания и контроля узлов учёта тепла, холодного и горячего водоснабжения на объектах водоканалов, коммунальных служб, котельных, ТЭЦ.

2. Установка и настройка программы

  В пакет сервисного программного обеспечения входит перечень файлов:

  • основные
    1. программа "km_5m_.exe" - программа считывания базы данных, распечатки и мониторинга параметров тепло/водопотребления;
    2. файлы "ArchKM.mdb, Report.rav" - файлы для распечатки параметров тепло/водопотребления;
    3. подкаталог "Images" содержит графические файлы с монтажными схемами для конфигурирования погружных приборов;
    4. подкаталог "Doc" с описанием сервисного программного обеспечения.
  • вспомогательные
    1. программа "PultU.exe" - дистанционный видеоконтроль и мониторинг параметров вычислителей КМ-5М, КМ-5-Б3(Б1), РМ-5-Б3(Б1);
    2. программы "AVRPC.exe", "HWIEW.exe" - технологические DOS-программы.
   Перечень основных файлов является достаточным для выполнения контроля и отчётов всех параметров тепло/водопотребления. Перечень вспомагательных файлов можно использовать для выполнения сервисных работ.
   Пакет сервисного программного обеспечения копируется в любую созданную рабочую директорию.

  При первом запуске программы "km_5m_.exe" необходимо выполнить настройку в окне "Настройка параметров", открываемом через пункт "Параметры" в меню "Настройка". Появится окно (Рис.2.1).

Рисунок 2.1. Окно "Настройка параметров".

  Приборы КМ,РМ могут подключаться к компьютеру через:

  • COM-порт, (в т.ч., виртуальный) по RS-232;
  • разъём Ethernet по локальной сети по TCP/IP.
Каждый тип подключения требует настройки.

2.1. Настройка подключения по RS-232

  1. Открыть вкладку "Связь" и выбрать RS-232 (Рисунок 2.1, поз.1). Отметить используемый COM-порт или несколько COM-портов, остальные "отключить". Возможен выбор виртуальных COM-портов, например, при подключении через интеграторы сети или автоматический преобразователь интерфейса АПИ-USB/RS485, КСПД-5.
  2. Флаг "Подбор скорости при неудачном соединении" рекомендуется оставить включённым, так как теплосчётчик/расходомер может быть настроен на скорость 19200 бод при считывании архивов БД. При установленном данном флаге программа автоматически "распознаёт" эту скорость. Иначе теплосчётчик/расходомер может не подключиться (если в ВУ была установлена скорость передачи данных 9600 бод), и потребуется его перезапустить, что не всегда представляется возможным. При считывании с УПД-8/16/32, АП-5-3 можно сбросить этот флаг, что ускорит процесс поиска прибора.
  3. Параметры "Время ожидания ответа" и "Количество попыток запросов" можно оставить без изменения, а редактировать в дальнейшем при необходимости. Например, при подключении к приборам по длинным линиям связи (RS-485) возможно потребуется увеличить этот параметр до 1000 (1 сек.). При подключении через КСПД-5 данный параметр может быть увеличен до 10000 (10 сек.).
  4. Флаг "Автоматический поиск прибора при запуске", рекомендуется установить при работе с одним теплосчётчиком/расходомером. В этом случае программа начинает поиск подключенного прибора сразу после запуска. Когда на линии подключены два и более приборов, "автоматический поиск прибора" рекомендуется отключить.
  5. Сохранить настройки можно, нажав кнопку "Запись". При этом закроется окно "Настройка параметров". Если нажать кнопку "Отмена", то окно закроется без сохранения параметров.

  1. Открыть вкладку "Связь" и выбрать TCP (Рисунок 2.1., поз.2). Окно примет вид: рисунок 2.2.

    Рисунок 2.2. Окно "Настройка параметров".

  2. В полях ввода "IP-адрес", "Порт", задать IP-адрес и номер порта устройства-посредника между ПК и теплосчётчиками/расходомерами. В качестве посредника может использоваться, например: интегратор сети, АПИ RS485/E.
  3. Параметры "Время ожидания ответа" и "Количество попыток запросов" можно оставить без изменения, а редактировать в дальнейшем при необходимости. Например, при подключении к приборам по длинным линиям связи.
  4. Флаг "Автоматический поиск прибора при запуске", рекомендуется установить только при подключении одного теплосчётчика/расходомера. В этом случае программа начинает поиск подключенного прибора сразу после запуска. Когда подключены два и более прибора, автоматический поиск рекомендуется отключить.
  5. Сохранить настройки можно, нажав кнопку "Запись". При этом закроется окно "Настройка параметров". Если нажать кнопку "Отмена", то окно закроется без сохранения параметров.

  1. Открыть вкладку "Мониторинг" (Рисунок 2.1., поз.3). Окно примет вид: рисунок 2.3.

    Рисунок 2.3. Окно "Настройка параметров".

  2. Параметр "Дискретность мониторинга" можно оставить без изменения. Он используется при мониторинге параметров тепло/водопотребления. Величину этого параметра можно изменять непосредственно перед запуском мониторинга в соответствии с рекомендациями, приведёнными в разделе: Мониторинг параметров тепло/водопотребления для теплосчётчиков/расходомеров

3. Загрузка архивных данных из теплосчётчиков/расходомеров в базу данных ПК

   Загрузка архивных данных из теплосчётчиков/расходомеров может осуществляться следующим образом:

  • при подключении одного или нескольких теплосчётчиков/расходомеров (объединенных в сеть RS-485) к последовательному порту ПК через адаптер периферии АП-5-2/3/4 или преобразователь интерфейса RS485/RS232, АПИ-5; АПИ-USB/RS485
  • по локальной сети и Internet через интегратор сети с разъёмом Ethernet, КСПД-5, АПИ-RS485/E;
  • по телефонной сети через модем;
  • посредством устройств переноса данных УПД-8/16/32, АП-5-3;
   Далее приведено более подробное описание загрузки архивных данных по каждому из описанных выше способов.

3.1. Загрузка архивных данных по сети RS-485

   Линия связи "RS-485" позволяет объединять в сеть несколько десятков теплосётчиков/расходомеров, подключённых к последовательному порту (COM) компьютера через АП-5-2/3/4, АПИ-5 и нуль-модемный кабель или через автоматический преобразователь интерфейса АПИ-USB/RS485.

При установленном флаге "Автоматический поиск прибора при запуске" (см. раздел "Установка и настройка программы") запустится процесс поиска подключённого теплосчётчика/расходомера. При успешном завершении поиска, в основном окне отобразится информация о подключённом приборе (Рисунок 3.3), после чего можно сразу переходить к пункту 4 данного раздела. При неуспешном завершении выдаётся сообщение "Устройство не обнаружено!". Тогда необходимо проверить исправность соединения и перейти к пункту 2 данного раздела.

    Рисунок 3.1 Основное окно программы.

  2. В поле "Порт" (Рисунок 3.1, поз.5) Выбрать требуемый COM-порт из списка, нажав стрелку вниз. Если подключен один теплосчётчик/расходомер, запустить процесс поиска теплочётчика/расходомера одним из способов:
    • активизировать поле ввода "№" (Рисунок 3.1, поз.3) и нажать клавишу "Ввод" (Enter);

       На экран выводится текущая информация о состоянии поиска. Процесс можно прервать в любой момент с помощью кнопки "Прервать".

При перезапуске вычислителя скорость устанавливается 9600 бод.
       Для уменьшения времени считывания архива целесообразно перейти на скорость 19200 бод. Но при этом прибор должен быть подключен к компьютеру через преобразователь интерфейса АПИ-5, АПИ-RS/USB, а также через КСПД-5 (см.раздел "Загрузка баз данных по локальной сети и Internet").        Аналогично можно перейти на скорость 9600 бод, если наблюдаются сбои по связи на скорости 19200 бод, или если теплосчётчик/расходомер подключён к диспетчерской системе ГИС ТБН.

  6. С целью экономии времени имеется возможность считывать не весь архив, а данные заданного периода. Для этого необходимо отметить "галочкой" позицию "Задать диапазон считывания:" (Рисунок 3.5, поз.3). Внизу появятся поля для ввода начальной и конечной даты выбираемого диапазона, которые следует заполнить. При снятии галочки данные поля с датами скрываются, и архив будет считываться полностью, независимо от значений скрытых полей дат.

       Нажать кнопку "Считать в файл" (Рисунок 3.5, поз.1). Если на ПК уже имеется считанный ранее файл архива данного прибора, то перед запуском процесса появляется предупреждение (Рисунок 3.6), и предлагаются на выбор три варианта:
    • Дополнить файл архива новыми записями (кнопка "Да");
    • Создать новый архив (кнопка "Нет").

3.3. Загрузка архивных данных с помощью модема

   Загрузка архивных данных из теплосчётчиков/расходомеров может выполняться практически через любой внешний модем, соответствующий требованиям:

  • Наличие точной инструкции по AT-командам.
  • В S-регистре должна запоминаться скорость COM-порта.
  • Возможность отключения режимов буферизации и автоматического управления потоками.
   Модем со стороны теплосчётчиков/расходомеров подключается к приборам через АП-5 или АПИ-5 (автоматический преобразователь интерфейса RS485 в RS232).
   Перед эксплуатацией модем со стороны теплосчётчиков/расходомеров должен быть соответствующим образом настроен. В документе "Как использовать модемы с теплосчётчиками КМ-5" приводится подробное описание и настроек различных модемов, которые сводятся к последовательности действий:
  • Подключиться к модему с персонального компьютера в режиме терминала на скорости 9600 бод без контроля чётности.
  • Командами модема (которые, к сожалению, могут различаться в разных моделях) выполнить:
    • отключить эхо и выдачу ответов;
    • отключить реакцию на сигналы DTR и RTS;
    • установить автоматический переход к соединению и в режим данных после определённого количества звонков;
    • отключить буферизацию MNP и всевозможные обработки ошибок;
    • отключить синхронизацию XON/XOFF и, возможно, другие похожие действия;
    • в некоторых модемах предусмотрена установка также фиксированной скорости в линии связи, что полезно использовать;
  • сохранить регистры модема в его энергонезависимой памяти в одном из профилей загрузки (AT&W0) и установить этот профиль в качестве профиля по умолчанию (ATY0).
   Проконтролировать установки по команде просмотра установок, обычно выводящей содержимое S-регистров и эквиваленты установок AT-команд в ОЗУ и в EEPROM. Команды просмотра установок могут различаться. Например, AT&V для модема Eline V1433VQE, ATI4 для модема 3COM US Robotics, AT&V для модема Zyxel Omni 56k.
   После настройки модема и подключении его к теплосчётчикам/расходомерам через АП-5 можно запускать программу "km_5m_.exe" на удалённом компьютере, к СОM-порту которого подключён другой модем. Выполнить последовательность действий по настройке сеанса связи:

   В процессе эксплуатации теплосчётчиков/расходомеров на объектах широко практикуется ежемесячное сохранение архивов данных в виде отдельных файлов. Данная процедура практикуется на источниках тепла: ТЭЦ, ТЭС, котельных, в тепловых сетях, водоканале, стоках.
   Программа "km_5m_.exe" предоставляет возможность экспортировать в файлы <.km5> почасовые, посуточные данные и данные событий/ошибок за любой период, задаваемый пользователем.

7. Настройка и считывание конфигурации теплосчётчиков/расходомеров

   По предварительному заказу теплосчетчик/расходомер может быть сконфигурирован под проект заказчика. В противном случае это необходимо сделать перед сдачей прибора в эксплуатацию. Кроме того, необходимость изменить конфигурацию может возникнуть в случае наращивания задач решаемых узлом учета (например, добавился канал учета ХВС), после монтажа дополнительных ППС и/или датчиков, а так же замене электронных блоков ППС.
   Для удобства работы с конфигурацией теплосчётчиков/расходомеров КМ-5М, КМ-5-Б1(РМ-5-Б1), КМ-5-Б3(РМ-5-Б3) в программе "km_5m_.exe" реализованы следующие функции:
  1. считывание конфигурации с прибора в файл с последующими просмотром и редактированием;
  2. просмотр и редактирование конфигурации, считанной ранее в файл;
  3. обнуление конфигурации (инициализация) в приборе;
  4. создание новой конфигурации;
  5. запись конфигурации, считанной/отредактированной/новой, в прибор;
  6. просмотр и распечатка карты параметров.

8. Тестирование вычислителя

   В программе "km_5m_.exe" реализованы функции тестирования связи ПК с вычислителем и тестирование Flash памяти вычислителя. Тестирование можно выполнять после успешного поиска программой подключённого прибора. 8.2. Тест Flash

   Данный тест проверяет Flash память вычислителя на отсутствие неисправных страниц. Процесс тестирования может занять несколько часов, поэтому рекомендуется запускать тест в случае крайней необходимости. Проверяются последовательно каждая страница. Перед проверкой содержимое страницы считывается и восстанавливается по завершении.
   Процесс тестирования можно прервать в любой момент без потери данных. При этом тестирование завершится после проверки текущей страницы.

   Последовательность действий:
  1. Открыть окно "Тест Flash" нажатием строки "Тест Flash" меню "Тесты" основного окна программы.
  2. Для экономии времени выбрать режим ускоренного тестирования, когда выполняется проверка части страниц в каждом секторе Flash памяти, что не обеспечивает полной гарантии обнаружения неисправности. Для ускоренного тестирования выбирается опция "Быстрое" (Рисунок 8.2, поз.3).

    Рисунок 8.2. Окно "Окно тестирования Flash".


  3. Нажать кнопку "Пуск" (Рисунок 8.2, Поз.1).
       Запустится процесс тестирования Flash. Кнопка примет вид "Останов". Линейка прогресса (Рисунок 8.2, поз.2) отображает текущий объём выполненного тестирования. Строки "Всего секторов", "Сектор", "Всего страниц" отображают текущее состояние. При обнаружении неисправной страницы выводится окно с сообщением о неисправности, и тестирование прекращается.
  4. По завершении тестирования выводится окно с сообщением об успешном прохождении теста. По нажатию кнопки "Ok" в сообщении окно теста связи примет исходный вид (Рисунок 8.2). Можно завершить тестирование, закрыв окно.
