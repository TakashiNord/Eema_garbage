1 Термины, определения, обозначения и сокращения

В настоящем документе приняты следующие термины, определения, обозначения и сокращения:

ODBC (англ. Open Database Connectivity) – программный интерфейс (API) доступа к базам данных, разработанный фирмой Microsoft.

БД – База Данных.

БДТИ – База Данных Технологической Информации РСДУ.

ТССО – Технологический сайт системного оператора.

 

2 Общие сведения

Название сервиса – «RSDU.Tsso».

2.1 Назначение сервиса

Сервис предназначен для автоматического получения плановых графиков собственных нужд от API технологического сайта АО «СО ЕЭС» ТССО и их записи в БДТИ РСДУ по заданному расписанию.

2.2 Необходимые условия

Перед переходом к действиям, описанным ниже в руководстве, должны быть выполнены следующие необходимые условия:

ü  установлен пакет .NET Framework 4.5;

ü  установлен пакет Visual C++ 2010 Redistributable Package;

ü  установлен пакет клиента Oracle 11.2;

ü  настроен источник ODBC для БДТИ РСДУ.

Необходимо наличие следующих исходных данных:

ü  ОС Microsoft Windows 2008 или выше.

2.3 Необходимые знания

Для выполнения действий, описанных ниже в руководстве, требуются следующие знания:

ü  имя пользователя ОС, имеющего права Администратора и его пароль на сервере установки сервиса;

ü  имя пользователя и пароль учетной записи РСДУ, которые используются сервисом для подключения к РСДУ;

ü  имя пользователя и пароль учетной записи администратора РСДУ, для выполнения операций настройки сервиса;

ü  наименование источников ODBC для доступа к БДТИ комплекса РСДУ.

2.4 Ссылки

Для настройки сервиса в БДТИ РСДУ следует ознакомиться со следующими документами:

ü  «Описание серверов и точек доступа в комплексе РСДУ. Руководство администратора»;

ü  «Описание структуры БД РСДУ. Руководство администратора»;

ü  «Создание нового сегмента сбора в разделе DA. Руководство администратора».

5 Администрирование сервиса

5.1 Регистрация и удаление сервиса в ОС Windows

5.1.1 Установка сервиса

Для установки сервиса необходимо войти в операционную систему Windows от имени пользователя, обладающего правами Администратора, и выполнить следующие действия:

1.    Скопировать дистрибутив сервиса (каталог Tsso) на целевой хост в каталог C:\RsduServices\.

При необходимости – отредактировать файл ServiceDefinition.xml (например, если требуется запустить несколько экземпляров сервиса).

2.    Выполнить установку сервиса с помощью скрипта «01. Install.bat»:

A.   открыть командную строку Windows от имени Администратора;

B.   перейти в каталог размещения сервиса и выполнить скрипт
«01. Install.bat».

После выполнения скрипта «01. Install.bat» сервис должен отображаться в консоли управления Службами Windows (Services).

Название сервиса Windows: RSDU> Сервис получения данных с ТССО.

3.    Выполнить конфигурирование сервиса с помощью приложения Tsso.Config.exe (см. раздел «Конфигурирование сервиса»).

4.    Выполнить настройку автоматического запуска и восстановления сервиса.

Для настройки автоматического запуска сервиса необходимо открыть свойства сервиса RSDU> Сервис получения данных с ТССО в консоли управления Службами Windows (Services), перейти во вкладку «Общие» и указать тип запуска – «Автоматически».

Настройка параметров восстановления при сбоях выполняется во вкладке «Восстановление». При этом необходимо указать для Первого, Второго и Последующих сбоев перезапуск через 1 минуту.

внимание!

Необходимо настроить запуск сервиса от имени пользователя, для которого был установлен сертификат ТССО. Для этого в окне свойств перейти во вкладку «Вход в систему» и настроить вход с учётной записью конкретного пользователя, при необходимости ввести пароль учётной записи.

 

5.1.1.1 Описание файла ServiceDefinition.xml

Описание сервиса, необходимое для регистрации сервиса в консоли управления Службы Windows, находится в файле ServiceDefinition.xml.

Файл ServiceDefinition.xml располагается в каталоге установки сервиса, по умолчанию – C:\RsduServices\Tsso\.

<?xml version="1.0" encoding="utf-16" ?>

<ServiceDefinition

xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

xmlns:xsd="http://www.w3.org/2001/XMLSchema"

xmlns="rsdu.ema">

<Name>RSDU.Tsso</Name>

<Display>RSDU> Сервис получения данных с ТССО</Display>

<Description>Сервис получения данных с ТССО</Description>

</ServiceDefinition>

Описание тэгов файла ServiceDefinition.xml:

ü  Name – уникальное в рамках ОС название сервиса (рекомендуется использовать латинские буквы и цифры без пробелов);

ü  Display – название сервиса, которое будет отображаться в консоли управления «Службы Windows» (допускается использование русских букв);

ü  Description – текстовое описание сервиса.

Файл ServiceDefinition.xml сконфигурирован для запуска одного экземпляра сервиса на одном сервере (ОС) с параметрами по умолчанию (типовая конфигурация).

При необходимости запуска нескольких экземпляров сервиса на одном сервере требуется для каждого экземпляра указать уникальные значения параметров Name и Display, а также разместить файлы сервиса в разных каталогах по числу запускаемых экземпляров.

5.1.2 Удаление сервиса

Для удаления сервиса необходимо войти в операционную систему Windows от имени пользователя, обладающего правами Администратора.

1.    Выполнить скрипт удаления сервиса «04. Uninstall.bat»:

A.   открыть командную строку Windows от имени Администратора;

B.   перейти в каталог размещения сервиса (по умолчанию – C:\RsduServices\Tsso) и выполнить скрипт «04. Uninstall.bat».

2.    В консоли управления Службами Windows (Services) убедиться в том, что в списке отсутствует сервис RSDU> Сервис получения данных с ТССО.

3.    При необходимости – удалить каталог размещения сервиса.

5.2 Запуск и останов сервиса

Запуск и останов сервиса выполняется следующими способами:

1.    с помощью консоли управления службами Windows (Services " Службы);

2.    из командной строки Windows.

5.2.1 Запуск/останов сервиса из командной строки

Запуск и останов сервиса из командной строки выполняется с помощью скриптов «02. Start.bat» и «03. Stop.bat» соответственно.

Для работы скриптов необходимы права Администратора.

Для запуска сервиса выполнить следующие шаги:

1.    Открыть командную строку Windows с правами Администратора.

Вызвать контекстное меню для приложения «Командная строка» («Пуск " Все программы " Стандартные " Командная строка») и выбрать пункт «Запуск от имени администратора».

2.    Перейти в каталог установки сервиса.

По умолчанию каталог установки: C:\RsduServices\Tsso\

3.    Запустить скрипт «02. Start.bat».

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

Для остановки сервиса необходимо выполнить следующие шаги:

1.    Открыть командную строку Windows с правами Администратора и перейти в каталог установки сервиса (см. выше);

2.    Запустить скрипт «03. Stop.bat».

5.2.2 Запуск/останов из консоли управления службами Windows

Для запуска или остановки сервиса с использованием консоли управления службами Windows необходимо выполнить следующие шаги:

1.    Открыть консоль управления службами Windows: «Пуск " Панель управления " Администрирование " Службы»;

2.    В списке служб найти и выделить службу «RSDU> Сервис получения данных с ТССО»;

3.    Нажать кнопку «Запуск службы» (для запуска службы);

4.    Нажать кнопку «Остановка службы» (для остановки службы).

Текущее состояние службы отображается в колонке «Состояние» в списке служб.

Работу сервиса можно проконтролировать по сообщениям в лог-файле (см. раздел «Контроль лог-файлов»).

5.3 Контроль лог-файлов

Лог-файлы сервиса располагаются в каталоге установки сервиса в папке logs. Например, C:\RsduServices\Tsso\logs\.

Сервис создаёт новый лог-файл каждые сутки (при наличии сообщений для записи в лог) с указанием даты в имени файла.

6 Конфигурирование сервиса

Для конфигурирования сервиса используется приложение Tsso.Config.exe, расположенное в каталоге установки сервиса (по умолчанию в каталоге C:\RsduServices\Tsso).

Окно конфигуратора содержит единственную вкладку – «Настройки». На ней перечислены все настройки сервиса по разделам. Все сделанные изменения в этой вкладке сохраняются автоматически.

Конфигурация сервиса хранится в XML файле settings.xml.

Определяет режим работы подсистемы логирования.

В режиме Debug выводится множество дополнительной информации о работе сервиса. Данный режим предназначен для отладки работы сервиса в случае возникновения ошибок.

Хранить файлы журнала дней

0-9999

По умолчанию: 0

Количество дней хранения файла журнала (лога).

Задаёт период хранения данных. По прошествии указанного количества дней журнал очищается. Если задано 0 – файлы журнала никогда не удаляются (не чистятся).

 

Приложение А. Формат файла settings.xml

Структура файла settings.xml:

<?xml version="1.0"?>

<Settings xmlns:i="http://www.w3.org/2001/XMLSchema-instance" xmlns="http://schemas.datacontract.org/2004/07/DBExchange.Core">

<LogMode>Normal</LogMode>

<RSDU_DB_AppName>Tsso.Service.exe</RSDU_DB_AppName>

<RSDU_DB_DSN>RSDU2</RSDU_DB_DSN>

<RSDU_DB_Login>tsso</RSDU_DB_Login>

<RSDU_DB_Password>rsdu_211</RSDU_DB_Password>

<StoreLogFilesDays>0</StoreLogFilesDays>

</Settings>

Описание тегов файла settings.xml:

ü  LogMode – режим работы подсистемы логирования;

ü  RSDU_DB_AppName – запускаемый модуль;

ü  RSDU_DB_DSN – имя DSN (ODBC);

ü  RSDU_DB_Login – имя процесса (пользователя);

ü  RSDU_DB_Password – пароль;

ü  StoreLogFilesDays – хранить файлы журнала, дней.
