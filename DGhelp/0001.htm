1 Термины, определения, обозначения и сокращения

В настоящем документе приняты следующие термины, определения, обозначения и сокращения:

ГОУ (Групповой Объект Управления) – совокупность объектов управления, оперативный персонал которых по команде или с разрешения диспетчера должен/может изменить производство/потребление реактивной мощности.

ГТП – группа точек поставки электроэнергии на оптовом рынке электроэнергии и мощности.

ДДГ (Доведенный Диспетчерский График) – диспетчерский плановый график на сутки вперед для второй неценовой зоны, доводится в виде 48 мгновенных значений мощности, заданных на середины получасовых интервалов. Для остальных моментов времени заданным мгновенным значением мощности является мощность, соответствующая точке на линии, состоящей из отрезков прямых, соединяющих вышеуказанные 48 точек. Определяет уточненный план нагрузки на сутки вперед генерирующих объектов.

Объем выработки по диспетчерскому графику по ДДГ – почасовой или получасовой объем выработки по заданному диспетчерскому графику. Является объем электроэнергии, определяемый для каждого часа или получаса заданным графиком мгновенной мощности генерации.

Объем выработки по ПДГ – плановое почасовое производство в ГТП генерации (ПДГ). Является объем электроэнергии, определяемый на указанный час как среднеарифметическое значение получасовой мощности. Для получасового интервала определяется половиной значения активной мощности, заданной на получасовом интервале.

ПДГ (Плановый (торговый) Диспетчерский График) – плановый график, заданный значениями активной мощности на получасовых интервалах, используемых для определения планового почасового объема производства электроэнергии на сутки вперед.

СДК (Стандартные Документируемые диспетчерские Команды) – команды, распоряжения, разрешения и сообщения, используемые при управлении режимами работы объектов генерации участников оптового рынка и внешними перетоками. Используются Системным оператором.

СН – собственные нужды, график потребления станции.

УДГ (Уточненный Диспетчерский График) – величина, полученная в результате изменения графика ДДГ путём отдачи команд системного оператора с указанием длительности и величины изменения ДДГ в отношении ГТП.

3.2.1 Папка

Элемент «Папка» группирует несколько элементов типа «Электростанция».

3.2.2 Электростанция

Элемент «Электростанция» описывает генерирующий энергообъект.

Элемент содержит ссылку на генерирующий объект из справочника «Структура объекта управления» БД РСДУ. Эта ссылка используется для определения диспетчерских центров, в ведении и управлении которых находятся ГТП генерирующего объекта.

Элемент содержит параметры:

ü  Pсум – суммарная активная мощность станции, МВт;

ü  Qсум – суммарная реактивная мощность станции, МВар;

ü  U – класс напряжения станции, кВ. Для каждого класса напряжения создаётся отдельный параметр;

ü  Uграф.макс – часовые значения максимума графика напряжения, кВ. Для каждого класса напряжения создаётся отдельный параметр;

ü  Uграф.мин – часовые значения минимума графика напряжения, кВ. Для каждого класса напряжения создаётся отдельный параметр.

Дерево информационной модели раздела может содержать несколько элементов типа «Электростанция».

3.2.3 Группа точек поставки

Элемент «Группа точек поставки» описывает генерирующую группу точек поставки электроэнергии (ГТП генерации) оптового рынка электроэнергии и мощности (ОРЭМ).

В отношении ГТП генерации регистрируются команды диспетчерского управления. В приложении «Диспетчерские графики» выбираются элементы типа «Группа точек поставки» при заполнении формы «Регистрация команд диспетчерского управления».

Элемент содержит параметры:

ü  Qмакс – максимально допустимое значение реактивной мощности ГОУ, МВАр. Рассчитывается на основе фактических значений активной мощности и диапазона регулирования реактивной мощности.

ü  Qмин – минимально допустимое значение реактивной мощности ГОУ, МВАр. Рассчитывается на основе фактических значений активной мощности и диапазона регулирования реактивной мощности;

ü  Uмакс – максимально допустимое значение напряжения, кВ. Определяется на основе СДК и графика напряжения;

ü  Uмин – минимально допустимое значение напряжения, кВ. Определяется на основе СДК и графика напряжения;

ü  ИВ0 – объем выработки электроэнергии по внешней инициативе системного оператора, МВт*ч;

ü  ИС – объем выработки электроэнергии по собственной инициативе, МВт*ч;

ü  Отклонение от УДГ – объём отклонения факта от плана, МВт*ч. Вычисляется по формуле: Отклонение от УДГ = Факт - УДГ;

Отклонение от УДГ в % – отклонение от УДГ, выраженное в процентах, %;

Дерево информационной модели раздела может содержать несколько элементов типа «Группа точек поставки» для станции.

3.2.4 Плановый график ДДГ

Элемент «Плановый график ДДГ» содержит параметры:

ü  ДДГ – значения планового графика ДДГ, МВт;

ü  ДДГ макс – значения максимума планового графика ДДГ, МВт;

ü  ДДГ мин – значения минимума планового графика ДДГ, МВт.

Для элемента «Группа точек поставки» может быть создан только один элемент типа «Плановый график ДДГ».

3.2.5 Плановый график ПДГ

Элемент «Плановый график ПДГ» содержит параметр:

ü  ПДГ – значение мощности выработки по графику ПДГ, МВт.

Параметр имеет настройку уставки «Контроль выработки».

Для элемента «Группа точек поставки» может быть создан только один элемент типа «Плановый график ПДГ».

3.2.6 Плановый график УДГ

Элемент «Плановый график УДГ» содержит параметры:

ü  УДГ – значения рассчитанного УДГ, МВт;

ü  УДГ (ИВ) – значения рассчитанного УДГ на основе команд с инициативой ИВ, МВт.

Параметры имеют настройки уставок: «Контроль мощности» и «Контроль выработки».

Для элемента «Группа точек поставки» может быть создан только один элемент типа «Плановый график УДГ».

3.2.7 Фактическая генерация

Элемент «Фактическая генерация» описывает группу генераторов, входящих в состав ГТП.

Элемент содержит параметры:

ü  Pсум – суммарная активная мощность по ГОУ, МВт;

ü  Qсум – суммарная реактивная мощность по ГОУ, МВАр.

Для элемента «Группа точек поставки» может быть создан только один элемент типа «Фактическая генерация».

3.2.8 Генератор

Элемент «Генератор» описывает генератор станции, входящий в состав ГТП.

Элемент типа «Генератор» содержит ссылку на соответствующее оборудование из справочника «Структура объекта управления» БД РСДУ.

Элемент содержит параметры:

ü  P – активная мощность, вырабатываемая генератором, МВт;

ü  Q – реактивная мощность, вырабатываемая генератором, МВАр.

Для элемента «Фактическая генерация» может быть создано несколько элементов типа «Генератор».

3.2.9 Собственные нужды (СН)

Элемент «Собственные нужды (СН)» описывает группу точек потребления электроэнергии (ГТП потребления) – собственное потребление электростанции.

Элемент не содержит параметры.

Дерево информационной модели раздела может содержать только один элемент типа «Собственные нужды (СН)» для конкретной станции.

3.2.10 Плановый график СН

Элемент «Плановый график СН» описывает плановое потребление собственных нужд станции.

Элемент содержит параметр:

ü  План СН – плановый график потребления СН, МВт;

Для элемента «Собственные нужды (СН)» может быть создан только один элемент типа «Плановый график СН».

3.2.11 Фактическое потребление

Элемент «Фактическое потребление» описывает фактическое потребление станции.

Элемент содержит параметры:

ü  Pсн – суммарная активная мощность, МВт;

ü  Qсн – суммарная реактивная мощность, МВАр.

Для элемента «Собственные нужды (СН)» может быть создан только один элемент типа «Фактическое потребление».

3.3 Создание узла

В левой части дерева описываются узлы со следующими типами: «Папка», «Электростанция», «ГТП», «Плановый график ДДГ», «Плановый график ПДГ», «Плановый график УДГ», «Плановый график СН», а также группирующие узлы для параметров фактического потребления и генерации.

внимание!

При создании узла автоматически создаются его параметры.

Исключение для узла «Электростанция» - его параметры необходимо создать вручную.

Элемент «Папка» можно создать в корневом узле раздела «Диспетчерские графики» или в другой папке. Папки служат для группировки станций, например, по территориальному признаку.

3.3.4 Создание плановых графиков

К плановым графикам относятся объекты типа «Плановый график ДДГ», «Плановый график ПДГ», «Плановый график УДГ» и «Плановый график СН».

Графики ДДГ, ПДГ и УДГ можно создать в узлах «Группа точек поставки». График СН можно создать только в узле «Собственные нужды».

Примечание 1. Нельзя создавать два графика одного типа внутри одного узла.

Примечание 2. Создание каждого типа планового графика внутри ГТП обязательно.

Элементы плановых графиков создаются аналогично элементу «Папка».

3.3.5 Создание элементов генерации

В узле «Группа точек поставки» должен быть описан единственный узел «Фактическая генерация». Он создается аналогично элементу «Папка».

Рисунок 6 – Пример создания элемента «Фактическая генерация»

Внутри узла «Фактическая генерация» создаются элементы «Генератор», которые соответствуют генераторам, входящим в состав описываемой ГТП.

Для создания генератора необходимо выбрать пункт «Создать» в контекстном меню узла «Фактическая генерация». В появившемся окне (Рисунок 7) требуется заполнить поля «Имя» и «Краткое имя», а также выбрать тип «Генератор». Далее необходимо нажать кнопку «…», расположенную справа от поля «Объект», для привязки параметра к оборудованию раздела структуры объекта и управления БД РСДУ.

Рисунок 7 – Пример создания элемента «Генератор»

В окне запущенного приложения «Настройка Базы Данных РСДУ – Adjust» необходимо выбрать элементы с типом «Генератор» и подтвердить выбор нажатием кнопки «Далее».

В окне «Источники» (Рисунок 8) необходимо описать и настроить канал, который должен ссылаться на параметр мощности генератора.

Рисунок 8 – Настройка источников для параметра

Если у генератора, выбранного из структуры объекта управления, есть параметр с типом «Суммарная мощность активная», то картридж автоматически создаст канал, настроенный на него. Если картридж не создаст канал автоматически, необходимо добавить канал вручную. Алгоритм добавления и настройки канала описан в разделе «Создание объекта фактического потребления».

Для завершения описания элемента генерации требуется нажать кнопку «Готово».

 

3.3.6 Создание элемента фактического потребления

В узле «Собственные нужды» должен быть создан единственный узел с типом «Фактическое потребление».

Рисунок 9 – Пример создания узла «Фактическое потребление»

Для создания необходимо выбрать пункт «Создать» в контекстном меню узла «Собственные нужды». В появившемся окне требуется заполнить поля «Имя» и «Краткое имя», а также выбрать тип «Фактическое потребление». По завершению редактирования свойств следует нажать кнопку «Далее». Появится окно, представленное на Рисунок 10.

Рисунок 10 – Окно настройки источников значений

Для того чтобы завершить создание элемента необходимо добавить и настроить источник значений. Для этого следует нажать кнопку «Добавить…» и в появившемся диалоге выбрать источник «Электрический режим».

В окне «Добавление канала» требуется заполнить имя канала и подтвердить ввод нажатием кнопки «ОК».

  

Рисунок 11 – Окно добавления канала источника значений

Далее следует выделить в дереве добавленный канал, после чего станет доступна кнопка «Свойства…», нажав на которую появится возможность выбора источника значений из раздела электрического режима (Рисунок 12).

Рисунок 12 – Настройка канала источника значений

В диалоге «Внешний источник значений» необходимо нажать кнопку «Добавить…». После этого откроется приложение «Настройка Базы Данных РСДУ – Adjust» с загруженным разделом электрического режима. В Adjust необходимо выбрать параметр, который должен обрабатываться в приложении «Диспетчерские графики» в качестве значения потребления для описываемой станции.

После завершения настройки канала следует подтвердить ввод нажатием кнопки «ОК».

Рисунок 13 – Задание текущего канала источника значений

В окне источников необходимо назначить текущим настроенный канал с помощью нажатия кнопки «Текущий».

Таким образом, у параметра можно описать несколько каналов, но приложение «Диспетчерские графики» будет запрашивать значения только из источника, который выбран текущим.

Для завершения описания фактического потребления требуется нажать кнопку «Готово».

4.2 Редактирование свойств параметров плановых графиков

Параметры имеют узлы плановых графиков: ДДГ, ПГД, УДГ, СН. Чтобы изменить свойства параметра, необходимо выбрать пункт «Свойства» контекстного меню параметра.

В появившемся окне свойств допускается изменение полей «Имя», «Краткое имя» и «Глобальный ИД».

Рисунок 15 – Окно изменения свойств параметров плановых графиков

Для параметров, описывающих плановый график УДГ, доступны дополнительные настройки:

ü  настройки диапазона контроля мощности ГТП – вкладка «Контроль мощности»;

ü  настройки уставок контроля выработки ГТП – вкладка «Контроль выработки».

Рисунок 16 – Вкладка «Контроль мощности» для УДГ

Вкладка «Контроль мощности» содержит два конфигурационных параметра:

ü  Допустимое отклонение от УДГ в МВт (L1) – определяет значение коридора

ü  Допустимое отклонение от УДГ в % (L2) – определяет значение коридора .

При расчете коридора контроля фактической мощности ГТП относительно УДГ значение L1 имеет более высокий приоритет, чем L2. Факт выхода значений активной мощности ГТП за пределы расчитанного коридора контроля будет выделено в приложении «Диспетчерские графики» специальной аварийной идикацией.

Вкладка «Контроль выработки» (Рисунок 17) содержит два конфигурационных параметра: «Предупредительный предел» и «Аварийный предел». Значения пределов задаются в процентах и определяют отклонения фактической выработки относительно плановой выработки по УДГ, при котором в приложении «Диспетчреские графики» отображается соответственно предупредительная и аварийная сигнализация для значений отклонения выработки.

Рисунок 17 – Вкладка «Контроль выработки» для УДГ

После редактирования свойств для принятия изменений необходимо нажать кнопку «ОК».

Для параметра, описывающего плановый график ПДГ, доступна настройка диапазона контроля выработки ГТП – вкладка «Контроль выработки».

Рисунок 18 – Вкладка «Контроль выработки» для ПДГ

Вкладка «Контроль выработки» содержит два конфигурационных параметра:

ü  Допустимое отклонение выработки в МВт*ч (P1) – определяет допустимое отклонение выработки от планового значения по ПДГ в абсолютных величинах (МВт*ч);

ü  Допустимое отклонение выработки в % (P2) – определяет допустимое отклонение выработки как процент от планового значения по ПДГ на заданном периоде.

Формула расчета выработки по собственной инициативе:

К объему выработки по внешней инициативе (ИВ) относится плановый объем выработки электроэнергии выше или ниже выработки относительно ДДГ.

Формула расчета объемов выработки по внешней инициативе:

(4)

Условия контроля выработки:

ü   – отклонение в допустимых пределах;

Для объектов с типами «Генератор» и «Фактическое потребление» создаются параметры, у которых обязательно настраиваются ссылки на исходные оперативные данные (вкладка «Источники»).

Чтобы изменить свойства параметра необходимо выбрать пункт «Свойства» контекстного меню для выделенного параметра.

На вкладке «Общие» допускается изменение полей «Имя» и «Краткое имя» либо задание глобального идентификатора.

Кроме того, на вкладке «Источники» имеется возможность добавления новых каналов или изменения свойств существующих.

 «Генератор» и «Фактическое потребление»

Приложение А. Изменение глобального идентификатора

Элементы информационной модели раздела содержат глобальный уникальный идентификатор «Глобальный ИД».

Этот идентификатор используется в задачах интеграции комплекса РСДУ с другими системами, а также при ретрансляции команд диспетчерского управления с РСДУ станции в РСДУ исполнительного аппарата. В этом случае глобальные идентификаторы всех элементов информационной модели одного экземпляра РСДУ должны совпадать с глобальными идентификаторами другого комплекса РСДУ.

внимание!

При ретрансляции команд диспетчерского управления дополнительно используются глобальные идентификаторы элементов справочников «Центры диспетчерского управления» и «Организационная структура».

 

Рисунок А.1 – Окно изменения глобального идентификатора

Форма «Изменение глобального идентификатора» позволяет изменить глобальный идентификатор вручную или сформировать новый уникальный идентификатор.

Для изменения идентификатора вручную необходимо в поле «Глобальный ИД» ввести строку в формате соответствующему стандарту RFC 4122. Если введенный идентификатор не соотвествует RFC 4122, то текст поля ввода будет окрашен в красный цвет, а кнопка «ОК» станет заблокированной.

Для генерации нового идентификтора необходимо нажать кнопку «Новый ИД».

Для завершения ввода глобального идентификатора следует нажать кнопку «ОК».
