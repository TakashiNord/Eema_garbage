1 Термины, определения, обозначения и сокращения

В настоящем документе приняты следующие термины, определения, обозначения и сокращения:

БДТИ – База Данных Технологической Информации РСДУ.

Предварительный диспетчерский график (ПДГ) – плановый график, заданный значениями активной мощности на получасовых интервалах, используемых для определения планового почасового объема производства электроэнергии на сутки вперед.

Раздел «Диспетчерские графики» – раздел БДТИ комплекса РСДУ, предназначенный для создания параметров с архивами данных диспетчерских графиков.

 

2 Общие сведения

2.1 Назначение приложения

Сервис получения ПДГ по электронной почте (pdg-server) получает по электронной почте предварительный диспетчерский график, разбирает и записывает его в архив РСДУ.

Сервис поставляется в виде службы Windows или Linux.

2.2 Необходимые условия

Перед переходом к действиям, описанным ниже в руководстве, должны быть выполнены следующие необходимые условия: установлена ОС Linux или Microsoft Windows:

ü  ОС Linux:

o   Ubuntu 16.04 или выше;

o   Debian 9 или выше;

o   openSUSE 15 или выше;

o   SUSE Enterprise Linux (SLES) 12 SP2 или выше.

ü  ОС Microsoft Windows:

o   Windows 7 SP1 или выше;

o   Windows Server 2012 R2+ или выше.

Необходимо наличие дистрибутива серверной части РСДУ:

ü  ОС Linux: LinuxNET-<version>-YYYYMMDD_HHMM.tar.bz2

ü  ОС Windows: WinNET-<version>-YYYYMMDD_HHMM.zip

2.3 Необходимые знания

Название сервиса: pdg-server.

Для выполнения действий, описанных ниже в руководстве, требуются следующие знания:

ü  имя пользователя ОС, имеющего права администратора и его пароль на сервере установки сервиса;

ü  параметры подключения к почтовому серверу, через который происходит получение ПДГ;

ü  параметры подключения к сервису доступа к архивам arch-webapi.

3 Установка и администрирование сервиса на ОС Linux

3.1 Установка сервиса

Установка сервиса проводится от имени пользователя с правами root. Если в используемой ОС Linux заблокирован вход от имени пользователя root, то необходимо использовать команду sudo для выполнения операций.

1.    Скопировать архив с дистрибутивами сервисов РСДУ для ОС Linux на целевой хост. Имя файла архива:

LinuxNET-<version>-YYYYMMDD_HHMM.tar.bz2

2.    Выполнить распаковку архива в домашний каталог пользователя:

tar -xvf LinuxNET-<version>-YYYYMMDD_HHMM.tar.bz2

3.    Создать рабочий каталог размещения сервисов РСДУ:

mkdir /usr/lib/ema

chown root:root /usr/lib/ema

mkdir /usr/lib/ema/services

chown root:root /usr/lib/ema/services

4.    Скопировать сервис pdg-server в рабочий каталог:

cp -R ./linux-x64/pdg-server /usr/lib/ema/services

5.    Создать каталог размещения конфигурационных файлов и выполнить копирование конфигурационных файлов по умолчанию.

Создание каталога размещения конфигурации:

mkdir /etc/ema/pdg-server

Копирование конфигурационных файлов по умолчанию:

cp /usr/lib/ema/services/pdg-server/conf/* /etc/ema/pdg-server/

6.    Выполнить редактирование конфигурационных файлов в каталоге /etc/ema/pdg-server/.

Описание конфигурационных файлов представлено в разделе «Конфигурирование сервиса».

7.    Выполнить регистрацию сервиса в systemd.

Скопировать скрипт описания сервисом для systemd:

cp /usr/lib/ema/services/pdg-server/systemd/pdg-server.service /etc/systemd/system

 

Выполнить настройку запуска сервиса при старте системы:

systemctl daemon-reload

systemctl enable pdg-server

3.2 Запуск и останов сервиса

Управление работой сервиса получения ПДГ по почте в ОС Linux осуществляется через стандартные команды systemd.

Запуск сервиса:

systemctl start pdg-server

Останов сервиса:

systemctl stop pdg-server

Перезапуск сервиса:

systemctl restart pdg-server

Контроль текущего состояния:

systemctl status pdg-server

3.3 Контроль лог файлов

Лог-файлы в ОС Linux сервиса располагаются в каталоге

/var/log/ema/pdg-server

Сервис ведет два лог файла:

ü  pdg-server.log – рабочий лог долговременного хранения,

ü  pdg-server_debug.log – отладочный лог краткосрочного хранения.

3.4 Удаление сервиса

Удаление сервиса выполняется от имени пользователя с правами root.

1.    Выполнить остановку сервиса:

systemctl stop pdg-server

2.    Выполнить удаление сервиса из числа зарегистрированных в systemd:

systemctl disable pdg-server

rm /etc/systemd/system/pdg-server.service

systemctl daemon-reload

3.    Удалить каталог размещения конфигурационных файлов:

rm –R /etc/ema/pdg-server

при необходимости удалить каталог /etc/ema/.

4.    Удалить рабочий каталог сервиса

rm –R /usr/lib/ema/services/pdg-server

При необходимости удалить каталоги /usr/lib/ema/services и /usr/lib/ema.

 

 

 

 

4 Установка и администрирование сервиса на ОС Windows

Необходимо войти в операционную систему Windows от имени пользователя, обладающего правами администратора.

1.    Распаковать архив WinNET-<version>-YYYYMMDD_HHMM.zip с дистрибутивом сервисов РСДУ во временный каталог.

2.    Скопировать каталог сервиса pdg-server из временного каталога в рабочий: c:\RSDUServices\pdg-server.

3.    Выполнить установку сервиса.

Открыть командную строку Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» («Пуск → Все программы → Стандартные → Командная строка») и выбрать пункт «Запуск от имени администратора».

Перейти в каталог размещения скриптов управления сервисом.

cd c:\RSDUServices\pdg-server\win-service

Выполнить скрипт:

install.bat

После выполнения скрипта install.bat сервис будет доступен в консоли управления Службами Windows (Services).

Название сервиса Windows: RSDU> Сервис получения ПДГ по почте.

4.    Выполнить редактирование конфигурационных файлов в каталоге
pdg-server\conf.

Описание конфигурационных файлов представлено в разделе «Конфигурирование сервиса».

5.    Выполнить настройку автоматического запуска и восстановления сервиса.

Для настройки автоматического запуска сервиса необходимо в консоли управления Службами Windows (Services) открыть свойства сервиса RSDU> Сервис получения ПДГ по почте, перейти во вкладку «Общие» и указать тип запуска «Автоматически».

Настройка параметров восстановления при сбоях выполняется во вкладке «Восстановление». При этом необходимо указать для Первого, Второго и Последующих сбоев перезапуск через 1 минуту.

4.1 Запуск и останов сервиса

Запуск и останов сервиса можно выполнить следующими способами:

1.    с помощью консоли управления службами Windows (Services → Службы);

2.    из командной строки Windows.

4.1.1 Запуск/останов сервиса из командной строки

Запуск и останов сервиса из командной строки выполняется с помощью скриптов start.bat и stop.bat, соответственно, которые расположены в каталоге
pdg-server\win-service.

Для выполнения скриптов start.bat и stop.bat требуются права администратора.

Для запуска сервиса необходимо выполнить следующие шаги:

1.    Открыть командную строку Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» («Пуск → Все программы → Стандартные → Командная строка») и выбрать пункт «Запуск от имени администратора».

2.    Перейти в каталог установки сервиса

cd c:\RSDUServices\pdg-server\win-service.

3.    Выполнить скрипт запуска start.bat.

Для остановки сервиса необходимо выполнить следующие шаги:

1.    Открыть командную строку Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» («Пуск → Все программы → Стандартные → Командная строка») и выбрать пункт «Запуск от имени администратора».

2.    Перейти в каталог установки сервиса

cd c:\RSDUServices\pdg-server\win-service.

3.    Выполнить скрипт останова stop.bat.

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

4.1.2 Запуск/останов из консоли управления службами Windows

Для запуска или остановки сервиса с использованием консоли управления службами Windows необходимо выполнить следующие шаги:

1.    Открыть консоль управления службами Windows «Пуск → Панель управления → Администрирование → Службы»;

2.    В списке служб выбрать RSDU> Сервис получения ПДГ по почте;

3.    Для запуска службы нажмите кнопку «Запуск службы»;

4.    Для остановки службы нажмите кнопку «Остановка службы».

Текущее состояние службы отображается в колонке «Состояние» в списке служб.

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

4.2 Контроль лог-файлов

Лог-файлы сервиса располагаются в подкаталоге logs, находящемся в каталоге установки сервиса. Файлы разбиваются по дате (дням) записи сообщений. Например, в C:\RSDUServices\pdg-server\logs.

Сервис ведет два лог файла:

ü  pdg-server.log – рабочий лог долговременного хранения;

ü  pdg-server_debug.log – отладочный лог краткосрочного хранения.

4.3 Удаление сервиса

Удаление сервиса выполняется от имени пользователя с правами администратора.

1.    Остановить сервис любым из указанных в настоящем руководстве методом.

2.    Открыть командную строку ОС Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» («Пуск → Все программы → Стандартные → Командная строка») и выбрать пункт «Запуск от имени администратора».

3.    Перейти в каталог установки сервиса

cd c:\RSDUServices\pdg-server\win-service.

4.    Выполнить скрипт останова uninstall.bat.

5.    В консоли управления Службами Windows (Services) проверить отсутствие в списке сервиса RSDU> Сервис получения ПДГ по почте.

6.    Удалить каталог установки сервиса, при необходимости.

 

5 Конфигурирование сервиса

Конфигурационные файлы сервиса в формате JSON:

ü  host_status.json – настройки модуля контроля состояния хоста системы резервирования (MASTER/SLAVE);

ü  email_reader.json – настройки подключения к серверу почты;

ü  pdgservice.json – настройки модуля, отвечающего за разбор Excel файлов и отправку данных сервису доступа к архивам arch-webapi;

ü  logging.json – настройки модуля ведения логов сервиса.

5.1 Файл host_status.json

Сервис получения ПДГ по почте поддерживает работу в инфраструктуре горячего резервирования хостов комплекса РСДУ5. Файл host_status.json содержит параметры модуля контроля состояния хоста (MASTER/SLAVE) подсистемы горячего резервирования. Формат файла:

{

  "HostStatusInformerOptions": {

    "DefaultHostStatus": "Master"

  },

 

  "HostStatusFromFileOptions": {

    "Enabled": false,

    "FilePath": "/var/run/ema/hoststate",

    "ReadFileInterval": 1

  }

}

Блок HostStatusInformerOptions – параметры информатора состояния хоста.

ü  DefaultHostStatus – статус хоста по умолчанию. Статус по умолчанию присваивается при старте сервиса.

Блок HostStatusFromFileOptions – настройки модуля контроля состояния хоста на основе файла. Подсистема горячего резервирования записывает текущее состояние хоста в файл, который располагается в памяти ОС или на диске.

ü  Enabled – определяет использование модуля контроля состояния хоста по данных из файла. true – модуль используется, false – модуль не используется;

ü  FilePath – каталог расположения файлов. Для ОС Linux по умолчанию /var/run/ema/hoststate;

ü  ReadFileInterval – интервал чтения файла. По умолчанию 1 секунда.

5.2 Файл email_reader.json

Содержит конфигурацию модуля получения файлов по почте. Формат файла:

{

    "EmailReaderOptions": {

 

        "Enabled": true,

 

        "EmailSettings": {

            "Protocol": "IMAP",

            "Host": "imap.mail.ru",

            "Port": "993",

            "UseSsl": true,

            "UserName": "pdgtest@mail.ru",

            "Password": "ema_2019",

            "UseCertificateValidation": false

        },

 

        "FileNameTemplates": [

            "24_pdg_vst_{ddMMyy}.xlsx"

        ],

 

        "ReadIntervalMin": 1

    }

}

EmailReaderOptions – основной блок, содержит всю конфигурацию модуля.

ü  Enabled – определяет использование модуля в составе сервиса. Значение всегда должно быть true – модуль используется.

ü  FileNameTemplates – список шаблонов наименований файлов, которые следует принять в обработку. Имя файла должно содержать дату, на которую записываются ПДГ. Формат и положение даты в шаблоне задается в фигурных скобках и соответствует формату, используемому в .NET для отображения объектов типа DateTime в виде строки. В текущем примере используется:

dd – день (2 цифры),

MM – месяц (2 цифры),

yy – год (2 последние цифры года);

ü  ReadIntervalMin – интервал опроса почты в минутах.

Блок EmailSettings – настройки подключения к почтовому серверу.

ü  Protocol – используемый протокол, поддерживаются IMAP и POP3;

ü  Host – имя хоста или IP-адрес сервера почты;

ü  Port – порт;

ü  UseSsl – true – использовать защищенное соединение, false – не использовать.

5.3 Файл pdgservice.json

Файл содержит настройки анализатора XLSX-файла и параметры подключения к сервису доступа к архивам arch-webapi:

{

  "ReceiveDataServiceOptions": {

 

    "Convertation": {

      "ConvertationFunc": "None"

    },

 

    "Parsers": [

      {

 

        "FileNameTemplates": [

          "24_pdg_vst_{ddMMyy}.xlsx"

        ],

 

        "SheetName": "Лист1",

        "ValueDirection": "Vertical",

        "ValueFirstCellNum": 8,

        "ValueCount": 24,

 

        "DgList": [

          {

            "Id": "1",

            "DgName": "ПДГ",

            "ColumnOrRowNum": 3,

            "TimeZoneOffsetMin": "300",

            "ValueIntervalSec": 3600,

 

            "Destinations": [

              {

                "Url": "http://192.168.12.116:8080",

                "Login": "admin",

                "Password": "password1",

                "TableId": 1320,

                "ParamId": 224,

                "TypeAlias": "GLT_ANALOG_OPT_CUR30"

              }

            ]

          }

        ]

      }

    ]

  }

}

ReceiveDataServiceOptions – основной блок, содержит всю конфигурацию модуля.

Блок Convertation – задаёт способ преобразования полученного графика в архив:

ü  ConvertationFunc – принимает значения:

o   None – преобразование не производится;

o   Linear – график преобразуется в архив типа GLT_ANALOG_OPT_CUR30 или GLT_ANALOG_OPT_CUR60 (см. параметр TypeAlias). Недостающие данные вычисляются с помощью линейной функции.

Блок Parsers – список конфигураций, описывающих расположение данных в файлах.

ü  FileNameTemplates – шаблоны имен файлов с указанием формата даты в фигурных скобках. Формат даты указывается с использованием спецсимволов, аналогично, как и для email_reader.json;

ü  SheetName – наименование листа с данными;

ü  ValueDirection – ориентация ряда со значениями:

o   Vertical – данные в колонках;

o   Horizontal – данные в строках;

ü  ValueFirstCellNum – номер ячейки с первым значением ПДГ. При размещении в колонках (Vertcal) – номер строки, в строках (Horizontal) – номер столбца;

ü  ValueCount – количество значений в ПДГ.

Блок DgList – список, описывающий диспетчерские графики:

ü  Id – номер графика в списке;

ü  DgName – наименование графика;

ü  ColumnOrRowNum – номер колонки (при вертикальном размещении) или строки (при горизонтальном) с данными;

ü  TimeZoneOffsetMin – смещение часового пояса относительно UTC в минутах;

ü  ValueIntervalSec – интервал времени между значениями мощности в полученном XLSX-файле в секундах, например:

o   3600 – для почасовых данных;

o   1800 – для получасовых данных.

 

Блок Destinations – список «приемников» данных ПДГ:

ü  Url – адрес сервиса доступа к архивам arch-webapi;

ü  Login – имя пользователя для подключения к arch-webapi;

ü  Password – пароль пользователя;

ü  TableId – идентификатор таблицы, содержащей параметр с идентификатором ParamId.

Идентификатор можно посмотреть в приложении Навигатор БД в свойствах узла «Технологическая информация → Контролируемые параметры → Диспетчерские графики» (Рисунок 1).

ü  ParamId – идентификатор параметра, к которому привязывается архив графика.

5.4 Файл logging.json

Файл содержит параметры записи лог-файлов сервиса. Ведение лог-файлов реализовано на основе библиотеки NLog. Файл по умолчанию содержит настройки записи основного лог-файла и расширенного с отладочной информацией.

Основной лог-файл хранится 3 месяца, отладочный лог – 7 дней.
