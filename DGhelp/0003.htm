БДТИ

База Данных Технологической Информации РСДУ

Раздел диспетчерских графиков (ДГ)

Раздел БДТИ содержащий справочник объектов для работы на оптовом рынке электроэнергии и мощности. Содержит плановые данные графики ДДГ, ПБР, УДГ

Раздел суточных графиков (HG)

Раздел БДТИ содержащий суточные графики, на основании которых сервер суточных графиков производит расчёт мгновенных значений в масштабе суток

ODBC

(англ. Open Database Connectivity) – программный интерфейс (API) доступа к базам данных, разработанный фирмой Microsoft

Общие сведения

 

Назначение сервиса

Сервис предназначен для переноса плановых данных (ДДГ, УДГ, ПБР) из раздела диспетчерских графиков в раздел суточных графиков БДТИ РСДУ, а также для отправки команд серверу суточных графиков на переинициализацию после переноса данных.

Необходимые условия

ОС Microsoft Windows 2008 или выше.

Установленный пакет .NET Framework 4.5.

Установленный пакет Visual C++ 2010 Redistributable Package.

Установленный пакет клиента Oracle 11.2.

Настроенный источник ODBC для БДТИ РСДУ.

Необходимые знания

Название сервиса – «RSDUTransferDGToHG».

Имя пользователя ОС, имеющего права администратора и его пароль на сервере установки сервиса.

Имя пользователя и пароль учетной записи РСДУ, используемой сервисом для подключения к РСДУ.

Имя пользователя и пароль учетной записи администратора РСДУ, для выполнения операций настройки сервиса.

Наименование источников ODBC для комплекса РСДУ.

Конфигурирование сервиса

 

Для конфигурирования сервиса используется приложение «TransferDGToHG.Config.exe», расположенное в каталоге установки сервиса (по умолчанию – каталог «C:\RsduServices\TransferDGToHG»).

Рисунок 2 – Приложение настройки сервиса (вкладка «Настройки»)

 

Окно конфигуратора содержит следующие вкладки:

Настройки – общие настройки сервиса, такие как параметры подключения к системам, периоды опроса, часовые пояса и т.д. Все сделанные изменения в этой вкладке сохраняются автоматически.

Связи – таблица соответствия объектов плановых данных из раздела диспетчерских графиков объектам в разделе суточных графиков. Для сохранения изменений в этой вкладке требуется нажать кнопку «Сохранить изменения».

Конфигурация сервиса хранится в XML файлах.

Файл settings.xml – файл конфигурации сервиса; содержит общие настройки сервиса (вкладка «Настройка»).

Файл links.xml – файл для хранения связей между объектами плановых данных и суточных графиков (вкладка «Связи»).

Описание структуры конфигурационных файлов приведено в Приложение 1: Формат конфигурационных файлов.

Для удобства выполнения настройки может потребоваться дополнительно запустить приложение «Навигатор БД» комплекса РСДУ (запустить «Панель оператора» и вызвать приложение «Навигатор БД»).

С помощью навигатора БД, например, можно будет получить информацию об идентификаторах объектов, используемых при настройке связей между объектами плановых данных и суточных графиков.

Администрирование сервиса

 

Регистрация и удаление сервиса в ОС Windows

Описание сервиса  (файл ServiceDefinition.xml)

Описание сервиса, необходимое для регистрации сервиса в консоли управления Службы Windows находится в файле ServiceDefinition.xml.

Файл ServiceDefinition.xml располагается в каталоге установки сервиса, по умолчанию – «C:\RsduServices\TransferDGToHG\».

<?xml version="1.0" encoding="utf-16" ?>

<ServiceDefinition

xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

xmlns:xsd="http://www.w3.org/2001/XMLSchema"

xmlns="rsdu.ema">

  <Name>RSDUTransferDGToHG</Name>

  <Display>RSDU> Сервис переноса ДГ в HG</Display>

  <Description>Сервис выполняет перенос диспетчерских графиков из раздела ДГ в раздел суточных графиков HG</Description>

</ServiceDefinition>

 

Описание тэгов файла ServiceDefinition.xml:

ü Name – уникальное в рамках ОС название сервиса; как правило, в названии сервиса используются латинские буквы и цифры;

ü Display – название сервиса, которое будет отображаться в консоли управления Службы Windows; допускается использование русских букв;

ü Description – текстовое описание сервиса.

Установка сервиса

Необходимо войти в операционную систему Windows пользователем с правами администратора.

1.     Скопировать дистрибутив сервиса (каталог TransferDGToHG) на целевой хост в каталог C:\RsduServices\.

При необходимости отредактировать файл ServiceDefinition.xml, например, для запуска нескольких экземпляров сервиса.

2.     Выполнить установку сервиса с помощью скрипта “01. Install.bat”.

Для этого необходимо открыть командную строку Windows от имени Администратора, перейти в каталог размещения сервиса и выполнить скрипт “01. Install.bat”.

После выполнения скрипта “01. Install.bat” сервис должен отображаться в консоли управления Службами Windows (Services).

Название сервиса Windows: «RSDU> Сервис переноса ДГ в HG».

3.     Выполнить конфигурирование сервиса с помощью приложения TransferDGToHG.Config.exe (см. раздел настоящей инструкции «Конфигурирование сервиса»).

4.     Выполнить настройку автоматического запуска и восстановления сервиса.

Для настройки автоматического запуска сервиса необходимо открыть свойства сервиса «RSDU> Сервис переноса ДГ в HG» в консоли управления Службами Windows (Services), перейти во вкладку «Общие» и указать тип запуска «Автоматически».

Настройка параметров восстановления при сбоях выполняется во вкладке «Восстановление». При этом необходимо указать для Первого, Второго и Последующих сбоев перезапуск через 1 минуту.

Удаление сервиса

Войти в операционную систему Windows пользователем с правами администратора.

1.     Выполнить скрипт удаления сервиса «04. Uninstall.bat».

Для этого необходимо открыть командную строку Windows от имени Администратора. Далее следует перейти в каталог размещения сервиса (по умолчанию C:\RsduServices\TransferDGToHG) и выполнить скрипт «04. Uninstall.bat».

2.     В консоли управления Службами Windows (Services) проверить отсутствие в списке сервиса «RSDU> Сервис переноса ДГ в HG».

3.     При необходимости удалить каталог размещения сервиса.

Запуск и останов сервиса

Запуск и останов сервиса можно выполнить следующими способами:

1.   С помощью консоли управления службами Windows (Services / Службы)

2.   Из командной строки Windows

Запуск/останов сервиса из командной строки

Запуск и останов сервиса из командной строки выполняется с помощью скриптов «02. Start.bat» и «03. Stop.bat» соответственно.

Для работы скриптов необходимы права администратора.

Для запуска сервиса необходимо выполнить следующие шаги:

1.   Открыть командную строку Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» (Пуск\Все программы\Стандартные\Командная строка) и выбрать пункт «Запуск от имени администратора».

2.   Перейти в каталог установки сервиса.

По умолчанию каталог установки: C:\RsduServices\TransferDGToHG\

3.   Запустить скрипт «02. Start.bat».

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

Для остановки сервиса необходимо выполнить следующие шаги:

1.   Открыть командную строку Windows с правами администратора и перейти в каталог установки сервиса (см. выше).

2.   Запустить скрипт «03. Stop.bat».

Запуск/останов из консоли управления службами Windows

Для запуска или остановки сервиса с использованием консоли управления службами Windows необходимо выполнить следующие шаги:

1.   Открыть консоль управления службами Windows.

     Пуск\Панель управления\Администрирование\Службы

2.   В списке служб найти и выделить службу «RSDU> Сервис переноса ДГ в HG».

3.   Для запуска службы нажмите кнопку «Запуск службы».

4.   Для остановки  службы нажмите кнопку «Остановка службы».

Текущее состояние службы отображается в колонке «Состояние» в списке служб.

Работу сервиса можно проконтролировать по сообщениям в лог-файле (см. Контроль лог-файлов).

Контроль лог-файлов

Лог-файл сервиса располагается в каталоге установки сервиса и называется TransferDGToHG.Service.log.

Например,

C:\RsduServices\TransferDGToHG\TransferDGToHG.Service.log

Раз в неделю сервис выполняет архивирование и очистку лог-фала. Архивный лог-файл сжимается и копируется в каталог archivelogs. В имени архивированного лог-файла содержится дата архивации.

Например,

C:\RsduServices\TransferDGToHG\archivelogs\ TransferDGToHG.Service_2016-01-07.log.gz

Приложение 1: Формат конфигурационных файлов

 

Структура файла settings.xml:

<Settings xmlns="http://schemas.datacontract.org/2004/07/TransferDGToHG.Core" xmlns:i="http://www.w3.org/2001/XMLSchema-instance">

<HgStoreDaysNumber>30</HgStoreDaysNumber>

<OffsetGraphMinutes>0</OffsetGraphMinutes>

<PreloadedDaysNumber>1</PreloadedDaysNumber>

<RSDU_DB_AppName>TransferDGToHG.Service.exe</RSDU_DB_AppName>

<RSDU_DB_DSN>RSDU2</RSDU_DB_DSN>

<RSDU_DB_Login>transdgtohg</RSDU_DB_Login>

<RSDU_DB_Password>rsdu_211</RSDU_DB_Password>

<TimerIntervalMinutes>1</TimerIntervalMinutes>

</Settings>

Описание тегов файла settings.xml:

HgStoreDaysNumber – Период хранения данных в разделе суточных графиков (дней);

OffsetGraphMinutes – Сдвиг графика (минут);

PreloadedDaysNumber – Количество загружаемых наперёд суток;

RSDU_DB_AppName – Запускаемый модуль;

RSDU_DB_DSN – Имя DSN (ODBC);

RSDU_DB_Login – Имя процесса (пользователя);

RSDU_DB_Password – Пароль;

TimerIntervalMinutes – Интервал опроса (минут).

 

Структура файла links.xml:

<DgToHgLinks xmlns="http://schemas.datacontract.org/2004/07/TransferDGToHG.Core" xmlns:i="http://www.w3.org/2001/XMLSchema-instance">

<Link>

<DGObjId>4</DGObjId>

<HGObjId>5021901</HGObjId>

</Link>

<Link>

<DGObjId>7</DGObjId>

<HGObjId>5021903</HGObjId>

</Link>

</DgToHgLinks>

Описание тегов файла links.xml:

Link – Объект связи;

DGObjId – Идентификатор объекта плановых данных в разделе диспетчерских графиков;

HGObjId – Идентификатор объекта суточного графика.
