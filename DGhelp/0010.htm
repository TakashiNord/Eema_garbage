Глоссарий

 

API

Интерфейс программирования приложений, интерфейс прикладного программирования.

(англ. Application Programming Interface]) — набор готовых классов, процедур, функций, структур и констант, предоставляемых приложением (библиотекой, сервисом) для использования во внешних программных продуктах.

GUID

Уникальный идентификатор информационных объектов справочников и журналов, используемый в РСДУ.

GUID комплекса РСДУ соответствует стандарту RFC4122.

RFC4122

Стандарт, определяющий правила формирования и использования UUID.

UUID

UUID (Universally Unique Identifier) — это стандарт идентификации.

Основное назначение UUID — это определить возможность распределённым системам уникально идентифицировать информацию без центра координации.

Наиболее распространённым использованием данного стандарта является Globally Unique Identifier (GUID) фирмы Microsoft.

КДУ

Команда диспетчерского управления

ГОУ

Групповой объект управления (ГОУ) - совокупность одной или нескольких групп точек поставки (ГТП), в отношении которой Системным оператором отдаются и фиксируются команды на изменение режима работы.

ГТП

Группа точек поставки (ГТП) – одна или несколько точек поставки, относящихся к одному узлу расчетной модели и/или к единому технологически не делимому энергетическому объекту и ограничивающих территорию, в отношении которой купля-продажа электрической энергии (мощности) на оптовом рынке осуществляется только данным участником, и используемых для определения и исполнения участником оптового рынка обязательств, связанных с поставкой и оплатой электрической энергии (мощности).

Группа точек поставки определяется Системным и Коммерческим операторами в соответствии с Положением о порядке получения статуса субъекта оптового рынка и ведения реестра субъектов оптового рынка электрической энергии и мощности.

СО (СО ЕЭС)

Системный Оператор Единой Энергосистемы Российской Федерации

ODBC

(англ. Open Database Connectivity) – программный интерфейс (API) доступа к базам данных, разработанный фирмой Microsoft

ОИК

Оперативный Информационный Комплекс

БДТИ

SQLite

Компактная встраиваемая реляционная база данных, используемая для хранения локальных данных.

Общие сведения

 

Назначение сервиса

Сервис предназначен для ретрансляции КДУ в темпе регистрации из комплекса РСДУ уровня станции в другой комплекс РСДУ, работающий на уровне филиалов или исполнительного аппарата генерирующей компании.

Необходимые условия

ОС Microsoft Windows 2008 или выше.

Установлен пакет .NET Framework 4.5.

Установлен пакет Visual C++ 2010 Redistributable Package.

Установлен пакет клиента Oracle 11.2.0.2.

Настроен источник ODBC для БДТИ локальной и удаленной РСДУ.

Выполнены обновления в БДТИ РСДУ:

ü 2013_12_02_DG_NEW;

ü 2014_03_04_J_DGSTATE;

ü 2015_10_08_ORG_STRUCTURE_AST;

ü 2015_09_15_DG_KDU_EXTENTION.

Необходимые знания

Название сервиса ретрансляции: «RSDU.INP\TransferKDUtoRSDU».

Имя пользователя ОС, имеющего права администратора и его пароль на сервере установки сервиса.

Имя пользователя и пароль учетной записи РСДУ, используемой сервисом передачи КДУ для подключения к локальной РСДУ.

Имя пользователя и пароль учетной записи РСДУ, используемой сервисом передачи КДУ для подключения к удаленной РСДУ.

Имя пользователя и пароль учетной записи администратора локальной и удаленной РСДУ, для выполнения операций настройки сервиса.

Наименование источников ODBC для локального и удаленного комплексов РСДУ.

Конфигурирование сервиса

 

Для конфигурирования сервиса используется приложение «TransferKDUtoRSDU.Config.exe», расположенное в каталоге установки сервиса (по умолчанию – каталог «C:\RsduServices\RSDU.INP\  TransferKDUtoRSDU»).

Рисунок 2 - Интерфейс приложения настройки сервиса вкладка «Настройки»

 

Окно конфигуратора содержит следующие вкладки:

Настройки – общие настройки сервиса, такие как параметры подключения к системам, периоды опроса, часовые пояса и т.д.

Рабочие переменные – это переменные, которые сохраняются при остановке сервиса и используются при его старте; рабочие переменные могут быть отредактированы.

Администрирование очереди передачи КДУ – содержит инструменты для контроля разбора очереди КДУ на передачу в удаленную РСДУ. Конфигурация сервиса хранится в файлах БД SQLite в каталоге db – config.db и work.db.

Файл config.db – файл конфигурации сервиса; содержит общие настройки сервиса (вкладка Настройка) и описание справочников интегрируемых систем (вкладка Связь справочников).

Файл work.db – файл для хранения рабочих переменных и очереди передачи КДУ.

Для удобства выполнения настройки необходимо дополнительно запустить приложение «Навигатор БД» комплекса РСДУ (запустить «Панель оператора» и вызвать приложение «Навигатор БД»). С помощью навигатора БД, например, можно будет получить информацию об идентификаторах объектов, используемых при настройке связей между системами.

Связь справочников

Для обеспечения целостности журналов КДУ локальной и удаленной РСДУ необходимо обеспечить идентичность справочников «Центры диспетчерского управления», «Организационная структура» и «Диспетчерские графики» обоих комплексов в рамках информационной модели локального комплекса (комплекса, который производит регистрацию КДУ).

Элементы справочников содержат поле «Глобальный ИД», которое определяет глобальный уникальный, для всех работающих на предприятии комплексов РСДУ, идентификатор элемента (GUID) в соответствии со стандартом RFC4122.

Идентичность справочников достигается за счет наличия в справочниках обоих комплексов одинаковых элементов – элементов с одинаковыми глобальными идентификаторами.

Перед запуском сервиса необходимо проверить идентичность справочников «Центры диспетчерского управления», «Организационная структура» и «Диспетчерские графики».

Для случая, когда нет необходимости повторения справочников «Центры диспетчерского управления» и «Организационная структура» в удалённой РСДУ, а также для обеспечения надежности выполнения процесса передачи КДУ в конфигурации сервиса предусмотрены следующие параметры, определяющие элементы справочников по умолчанию:

ü Уникальный идентификатор (UUID) диспетчерского центра по умолчанию;

ü Уникальный идентификатор (UUID) персонала по умолчанию;

Эти параметры используются в случае, если в удалённом комплексе РСДУ отсутствуют элементы справочника «Центров диспетчерского управления» и «Организационная структура», которые были указаны при регистрации КДУ (раздел «КДУ отдал» формы регистрации команды приложения «Диспетчерские графики») в локальном комплексе РСДУ.

Для заполнения этих параметров необходимо выполнить следующие шаги:

1.     На АРМ удалённого комплекса РСДУ необходимо запустить приложение «Панель оператора» и осуществить вход в систему от имени пользователя с правами администратора.

2.     Запустить приложение «Навигатор БД» и перейти в раздел «Служебная информация\Справочники».

3.     В разделе «Центры диспетчерского управления» необходимо выбрать элемент, который будет использоваться по умолчанию, и вызвать его свойства.

Администрирование сервиса

 

Регистрация и удаление сервиса в ОС Windows

Описание сервиса  (файл ServiceDefinition.xml)

Описание сервиса, необходимое для регистрации сервиса в консоли управления Службы Windows находится в файле ServiceDefinition.xml.

Файл ServiceDefinition.xml располагается в каталоге установки сервиса, по умолчанию – «C:\RsduServices\RSDU.INP\TransferKDUtoRSDU\».

<?xml version="1.0" encoding="utf-16" ?>

<ServiceDefinition

xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

xmlns:xsd="http://www.w3.org/2001/XMLSchema"

xmlns="rsdu.ema">

  <Name>RSDUINPTransferKDUtoRSDU</Name>

  <Display>RSDU.INP> Сервис передачи КДУ в РСДУ</Display>

  <Description>Сервис передачи КДУ в удаленную РСДУ. Входит в состав подсистемы RSDU.INP комплекса РСДУ.</Description>

</ServiceDefinition>

 

Описание тэгов файла ServiceDefinition.xml:

ü Name – уникальное в рамках ОС название сервиса; как правило, в названии сервиса используются латинские буквы и цифры;

ü Display – название сервиса, которое будет отображаться в консоли управления Службы Windows; допускается использование русских букв;

ü Description – текстовое описание сервиса.

Установка сервиса

Необходимо войти в операционную систему Windows пользователем с правами администратора.

1.     Скопировать дистрибутив сервиса (каталог RSDU.INP\TransferKDUtoRSDU) на целевой хост в каталог C:\RsduServices\RSDU.INP.

При необходимости отредактировать файл ServiceDefinition.xml.

ВАЖНО!!!

Необходимо выполнить редактирование файла ServiceDefinition.xml в случае запуска нескольких экземпляров сервисов на передачу КДУ. Например, для обеспечения параллельной передачи КДУ из РСДУ уровня станции в комплексы РСДУ исполнительного аппарата и филиала генерирующей компании.

2.     Выполнить установку сервиса с помощью скрипта “01. Install.cmd”.

Для этого необходимо открыть командную строку Windows от имени Администратора, перейти в каталог размещения сервиса и выполнить скрипт “01. Install.cmd”.

После выполнения скрипта “01. Install.cmd” сервис должен отображаться в консоли управления Службами Windows (Services).

Название сервиса Windows: «RSDU.INP> Сервис передачи КДУ в РСДУ».

3.     Выполнить конфигурирование сервиса с помощью приложения TransferKDUtoRSDU.Config.exe (см. раздел настоящей инструкции «Конфигурирование сервиса»).

4.     Выполнить настройку автоматического запуска и восстановления сервиса.

Для настройки автоматического запуска сервиса необходимо открыть свойства сервиса «RSDU.INP> Сервис передачи КДУ в РСДУ» в консоли управления Службами Windows (Services), перейти во вкладку «Общие» и указать тип запуска «Автоматически».

Настройка параметров восстановления при сбоях выполняется во вкладке «Восстановление». При этом необходимо указать для Первого, Второго и Последующих сбоев перезапуск через 1 минуту.

Удаление сервиса

Войти в операционную систему Windows пользователем с правами администратора.

1.     Выполнить скрипт удаления сервиса «04. Uninstall.cmd».

Для этого необходимо открыть командную строку Windows от имени Администратора. Далее следует перейти в каталог размещения сервиса (по умолчанию C:\RsduServices\RSDU.INP\ TransferKDUtoRSDU) и выполнить скрипт «04. Uninstall.cmd».

2.     В консоли управления Службами Windows (Services) проверить отсутствие в списке сервиса «RSDU.INP> Сервис передачи КДУ в РСДУ».

3.     При необходимости удалить каталог размещения сервиса.

Запуск и останов сервиса

Запуск и останов сервиса можно выполнить следующими способами:

1.   С помощью консоли управления службами Windows (Services / Службы)

2.   Из командной строки Windows

Запуск/останов сервиса из командной строки

Запуск и останов сервиса из командной строки выполняется с помощью скриптов «02. Start.cmd» и «03. Stop.cmd» соответственно.

Для работы скриптов необходимы права администратора.

Для запуска сервиса необходимо выполнить следующие шаги:

1.   Открыть командную строку Windows с правами администратора.

Вызвать контекстное меню для приложения «Командная строка» (Пуск\Все программы\Стандартные\Командная строка) и выбрать пункт «Запуск от имени администратора».

2.   Перейти в каталог установки сервиса.

По умолчанию каталог установки: C:\RsduServices\RSDU.INP\TransferKDUtoRSDU

3.   Запустить скрипт «02. Start.cmd».

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

Для остановки сервиса необходимо выполнить следующие шаги:

1.      Открыть командную строку Windows с правами администратора и перейти в каталог установки сервиса (см. выше).

2.      Запустить скрипт «03. Stop.cmd».

Запуск/останов из консоли управления службами Windows

Для запуска или остановки сервиса с использованием консоли управления службами Windows необходимо выполнить следующие шаги:

1.   Открыть консоль управления службами Windows.

     Пуск\Панель управления\Администрирование\Службы

2.   В списке служб найти и выделить службу «RSDU.INP> Сервис передачи КДУ в РСДУ».

3.   Для запуска службы нажмите кнопку «Запуск службы».

4.   Для остановки  службы нажмите кнопку «Остановка службы».

Текущее состояние службы отображается в колонке «Состояние» в списке служб.

Работу сервиса можно проконтролировать по сообщениям в лог-файле (см. Контроль лог-файлов).

Управление рабочими переменными

Рабочие переменные сервиса – это переменные, которые сервис сохраняет в файл в темпе их изменения в процессе работы и читает из файла при запуске.

ВАЖНО!

Редактирование переменных возможно только при остановленном сервисе, поскольку сервис выполняет чтение переменных из файла при старте, и в процессе работы осуществляет запись переменных в файл.

Для изменения значения переменной выберите ячейку колонки «Значение», введите новое значение и нажмите Enter или переместите фокус на другой элемент или строку.

Кнопка «Обновить» производит обновление значений переменных из файла.

Администрирование очереди передачи КДУ

Вкладка «Администрирование очереди передачи КДУ» отображает элементы очереди на передачу, состояние ее элементов и сообщения об ошибках.

Рисунок 6 - Интерфейс консоли администрирования очереди на передачу КДУ

 

Описание элементов управления:

Рисунок 7 - Элементы управления очередью передачи  КДУ

 

Поле «Статус» определяет фильтр элементов очереди по текущему состоянию.

Возможные варианты поля «Статус»:

ü ALL – все записи очереди.

ü DONE – успешно обработанные (переданные) КДУ.

ü ERROR – КДУ, при передачи которых возникла ошибка.

ü NEW – новые КДУ в очереди, ожидающие передачу.

Поле «Дата начала» и «Дата окончания» задают фильтр выбора элементов очереди по времени поступления в очередь – фильтр по полю «Дата» таблицы.

Кнопка «Выбрать» производит выбор элементов очереди по заданному полями «Статус», «Дата начала» и «Дата окончания» фильтру.

Кнопка «Удалить» производит удаление выбранного элемента очереди в таблице.

Кнопка «Изменить» открывает окно изменения статуса и сообщения об ошибке элемента очереди.

Кнопка «Очистить» позволяет выполнять очистку старых обработанных записей очереди. Используется для уменьшения размера файла Work.db.

Описание полей таблицы отображения элементов очереди:

ü Id – идентификатор элемента очереди;

ü IdParent – ссылка на другой элемент очереди; используется при передачи операций над зарегистрированными КДУ – операция КДУ;

ü Дата – дата поступления КДУ в очередь на передачу;

ü Состояние – текущее состояние элемента в очереди; возможны состояния: NEW – новый элемент, DONE – успешно обработанный элемент, ERROR – элемент, при обработке которого возникла ошибка;

ü Id объекта (локальный) – идентификатор КДУ в локальной РСДУ;

ü Id объекта (внешний) – идентификатор КДУ в удаленной РСДУ (совпадает с Id объекта (локальный));

ü Текст – текстовое представление КДУ.

Для удобства контроля процесса передачи КДУ под таблицей расположены два окна, отображающих текстовое представление КДУ и в виде XML.

Описание тэгов и атрибутов XML представления:

ü Command – описание команды из справочника команд (таблица DG_KDU_COMMAND);

ü Initiative – описание инициативы из таблицы DG_KDU_INITIATIVE;

ü Cause – описание причины отклонения из таблицы DG_KDU_CAUSE;

ü CauseCommentTmpl – описание шаблона комментария к причине отклонения из таблицы DG_KDU_COMMENT_TMPL;

ü GUID – глобальный идентификатор КДУ в журнале;

ü Gou – описание ГОУ из таблицы DG_GROUPS;

ü dt1970_start и dt1970_end – даты начала и окончания КДУ в формате DT1970;

ü Value – значение (определяет значение МВт, указанное при регистрации КДУ);

ü IsDeleted – флаг, определяющий удаление КДУ;

ü dt1970_create – дата регистрации КДУ в РСДУ в формате DT1970;

ü IdUserCreate – идентификатор пользователя РСДУ, зарегистрировавшего КДУ;

ü dt1970_modify – дата изменения КДУ в формате DT1970;

ü IdUserModify – идентификатор пользователя РСДУ, выполнившего изменение КДУ;

ü dt1970_receive – дата получения КДУ в формате DT1970;

ü Commentary – комментарий;

ü SourceCnt – описание диспетчерского центра из таблицы AST_CNT, отдавшего КДУ;

ü SourceDisp – описание персонала из таблицы AST_PERSONNEL, отдавшего КДУ;

ü TextCommand – текстовое представление КДУ.

Контроль лог-файлов

Лог-файл сервиса располагается в каталоге установки сервиса и называется TransferKDUtoRSDU.Service.log.

Например,

C:\RsduServices\RSDU.INP\TransferKDUtoSK2007\ TransferKDUtoRSDU.Service.log

Раз в неделю сервис выполняет архивирование и очистку лог-фала. Архивный лог-файл сжимается и копируется в каталог archivelogs. В имени архивированного лог-файла содержится дата архивации.

Например,

C:\RsduServices\RSDU.INP\TransferKDUtoRSDU\archivelogs\ TransferKDUtoRSDU.Service_2016-01-07.log.gz
