Мониторинг выполнения диспетчерских графиков

 

Назначение программы

Приложение предназначено для контроля режима работы генерирующих объектов в условиях оптового рынка электроэнергии и мощности (ОРЭМ).

Приложение может быть сконфигурировано для работы как на отдельной станции с одной или несколькими ГТП, так и в Исполнительном Аппарате Генерирующей Компании.

В случае установки приложения в исполнительном аппарате Генерирующей Компании приложение обеспечивает контроль выполнения диспетчерского графика каждой станцией, включая прием и исполнение станциями команд КДУ.

Системные требования

ü  ОС Microsoft Windows 2000 или выше;

ü  Установленный пакет .NET Framework 4.0;

ü  Установленный клиент РСДУ5;

ü  Разрешение экрана монитора не менее 1280х800.

Определения, Акронимы и Сокращения

СО – системный оператор;

ГТП – группа точек поставки;

ГОУ – групповой объект управления;

ДДГ – Доводимый Диспетчерский График;

КДУ – Команда диспетчерского управления;

УДГ – Уточненный Диспетчерский График, величина, полученная в результате изменения графика ДДГ путём отдачи команд СО в отношении ГТП;

ФАКТ генерации – фактические показания генерации станции;

СН – собственные нужды, график потребления станции.

Общие сведения

Приложение обрабатывает график ДДГ для каждой описанной ГТП станции, полученной от Системного Оператора. Прием ДДГ может осуществляться как автоматически (с помощью сервиса переноса данных), так и вручную (с помощью программы импорта данных из XLS файла). График помещается для долговременного хранения в раздел диспетчерских графиков базы данных РСДУ5.

Данные о значениях мгновенной мощности генераторов, на основе которых происходит расчет суммарных значений ГТП, и текущие интегральные величины приложение получает от сервера данных комплекса РСДУ.

Приложение выполняет следующие функции:

ü  просмотр информации о выполнении плана генерации на текущие сутки (на любую доступную дату) в виде графика и в табличном варианте;

ü  просмотр информации о выполнении плана потребления на текущие сутки (на любую доступную дату) в виде графика и в табличном варианте;

ü  просмотр графика генерации/потребления в отдельном окне в темпе реального времени;

ü  просмотр текущих показателей станции и рассчитанных рекомендаций для выполнения плана на текущий час;

ü  поддержка ввода КДУ и пересчета графика генерации;

ü  расчет рабочего коридора, в пределах которого допускается отклонение графика;

ü  расчет отклонений от плановых графиков, как в абсолютных, так и относительных величинах;

ü  сигнализация об отклонении от графика.

Для работы с программой необходимо наличие следующих прав у пользователя комплекса:

ü  «Чтение» и «Вход в ОИК» для подсистемы «Базовая информация комплекса»;

ü  «Чтение» для подсистемы «Оперативный информационный комплекс»;

ü  «Чтение» для подсистемы «Подсистема контроля и резервирования»;

ü  «Чтение» для подсистемы «Пользователи»;

ü  «Чтение» для подсистемы «Серверы архивов»;

ü  «Чтение» для подсистемы «Средства отображения».

Запуск программы

Программу можно запустить несколькими способами:

1.     Запустить на выполнение файл DGViewer.exe из системного каталога РСДУ5 (по умолчанию – R:\Bin\).

2.     Открыть через ярлык на рабочем столе.

3.     Выбрать из списка прикладных программ в панели оператора (Appbar) пункт «Диспетчерские графики».

Приложение DGViewer можно запустить с дополнительными параметрами, которые влияют на отображение информации в клиенте. Подробности представлены в разделе настоящего документа «Приложение 3. Параметры запуска приложения».

После описания необходимой информации в базе данных с помощью «Навигатора БД» приложение автоматически сформирует рабочее пространство для работы с несколькими ГТП.

Окно «Контроль объёмов»

Окно «Контроль объемов» содержит таблицу с данными о значениях объёмов выработки или потребления по плану, по факту и разницу между ними за сутки, выбранные в панели инструментов окна ГТП.

Рисунок 4 – Окно «Контроль объёмов» для ГТП генерации за текущие сутки

 

Для ГТП генерации таблица содержит следующие поля:

ü Время – диапазон времени, соответствующий получасовому или часовому интервалу;

ü Факт – объём фактически выработанной электроэнергии;

ü Откл. – объём отклонения факта от плана, вычисляемый по формуле: Откл. = Факт – УДГ;

ü Откл. % – значение отклонения Откл., выраженное в процентах;

ü ПДГ – плановый объем выработки электроэнергии по ПДГ;

ü ДДГ – плановый объём выработки электроэнергии по ДДГ;

ü УДГ – объём выработки электроэнергии по УДГ, с учетом принятых команд диспетчерского управления;

ü УДГ' – плановый объем выработки электроэнергии по ДДГ с учетом внешних инициатив;

ü ИС - объем выработки электроэнергии по собственной инициативе;

ИС- - значение ИС по модулю, если ИС < 0;

ИС+ - значение ИС, если ИС > 0;

ü ИВ0 – объем выработки электроэнергии по внешней инициативе системного оператора;

ИВ0- - значение ИВ0 по модулю, если ИВ0<0;

ИВ0+ - значение ИВ0, если ИВ0>0;

ü Доп.откл. – допустимое отклонение выработки по собственной инициативе.

Для ГТП собственных нужд таблица содержит следующие поля:

ü Время – диапазон времени, соответствующий получасовому или часовому интервалу;

ü Факт – объём фактически потреблённой электроэнергии;

ü Откл. – объём отклонения факта от плана, вычисляемый по формуле: Откл. = Факт – План;

ü Откл. % – значение отклонения Откл., выраженное в процентах;

ü План – плановый объём потребления электроэнергии.

Рисунок 11 – Окно «Команды диспетчерского управления»

 

Окно содержит список команд диспетчерского управления для текущего ГТП, действие которых распространяется на период выбранных суток. Журнал КДУ представлен в табличной форме, где каждая команда отображается в виде отдельной строки.

Таблица содержит следующие поля:

ü КДУ – краткое текстовое описание команды;

Серым шрифтом курсивного начертания отображается команда, зарегистрированная за прошедшие сутки и действие которой распространяется на текущие (выбранные) сутки (кроме команды «Работать по ДГ»).

Колонки «Получена», «КДУ», «Начало» и «Окончание» жестко зафиксированы и не перемещаются при использовании горизонтального скроллинга.

ü   «Регистрация новой КДУ» – выполнить регистрацию новой КДУ в журнале;

ü  перемещение графика осуществляется с помощью нажатой левой клавиши и одновременным перемещением мыши.

Организация рабочего пространства

Главное окно приложения состоит из отдельных окон для каждой ГТП. Окна ГТП располагаются одно за другим, в виде вкладок. С помощью панели переключения ГТП можно выбирать текущую вкладку. Каждое окно ГТП в свою очередь содержит несколько дочерних информационных окон. Любое окно может быть вынесено из главного окна приложения и расположено в произвольном месте рабочего стола операционной системы. Также любое окно может быть перемещено в пределах главного окна, сгруппировано с другими окнами или скрыто. Благодаря этому пользователь может настроить для себя максимально комфортное рабочее место.

Все выполненные настройки автоматически сохраняются, поэтому пользователю не требуется осуществлять повторную настройку при следующем запуске приложения.

Команды диспетчерского управления

Приложение поддерживает функции регистрации и удаления команд диспетчерского управления. Журнал команд сохранятся в БД комплекса РСДУ.

При регистрации или удалении КДУ приложение автоматически выполняет расчет уточненного диспетчерского графика (УДГ), допустимого коридора активной мощности и плановых объемов выработки электроэнергии для ГТП.

Примечание. Комплекс РСДУ может автоматически выполнять ретрансляцию журнала КДУ в темпе регистрации/удаления команд в РСДУ или другие системы уровня исполнительного аппарата (филиала) генерирующей компании. Для этого необходимо выполнить настройку и запуск дополнительных интеграционных модулей РСДУ.

Регистрация команды диспетчерского управления

Приложение «Диспетчерские графики» поддерживает работу с группами «Команды по изменению активной нагрузки» и «Специальные стандартные документированные диспетчерские команды» в соответствии с регламентом СО ЕЭС «Приложением №1 Порядка отдачи и регистрации стандартных документированных диспетчерских команд, распоряжений…».

Для регистрации КДУ необходимо нажать на кнопку «Регистрация новой КДУ» () на панели инструментов окна «Команды диспетчерского управления» ГТП или из общего окна «Журнал КДУ» и заполнить форму «Регистрация КДУ».

Рисунок 30 – Регистрация новой команды диспетчерского управления

 

Форма «Регистрация КДУ» содержит следующие поля:

ü  Объект – ГОУ (ГТП), относительно которой выдана команда;

ü  Команда – команда диспетчерского управления;

ü  Значение «N» МВт – значение активной мощности;

ü  Инициатива – определяет инициативу выдачи команды;

ü  Время получения команды – время доведения команды диспетчером СО ЕЭС;

ü  Время окончания выполнения; – дата и время, определяющее окончание команды;

Поле «Диспетчерский пункт» заполняется информацией из настройки «Ведение», расположенной на вкладке «Принадлежность», в свойствах элемента станции в разделе «Структура объекта управления». Если привязка к объекту не осуществлена, то отображается вся иерархия центров диспетчерского управления. Информацию о настройке ведения/управления объекта станции можно прочитать в описании картриджа «Диспетчерские графики» DBDispGraph.

После заполнения формы «Регистрация КДУ» необходимо нажать кнопку OK. При этом приложение автоматически выполнит запись новой команды в журнал КДУ, произведет расчет уточненного диспетчерского графика (УДГ), допустимого коридора УДГ, произведет перерасчет плановых объемов производства электроэнергии и установит новые уставки для контроля режимов по мгновенной активной мощности и объемам выработки электроэнергии.

Зарегистрированная команда диспетчерского управления будет автоматически отображена в окне «Команды диспетчерского управления» соответствующей ГТП.

Обновление отображения журнала команд в окне «Журнал КДУ» выполняется вручную при нажатии на кнопку «Показать».

Удаление команды диспетчерского управления

Приложение «Диспетчерские графики» позволяет выполнять удаление зарегистрированных команд из журнала КДУ.

Примечание. Для удаления КДУ у пользователя должно быть установлено право Настройка для подсистемы «Оперативный информационный комплекс».

Примечание. Удаление команд из журнала КДУ разрешено для команд, зарегистрированных в течение последних 24 часов. Удаление более поздних КДУ запрещено.

Для удаления зарегистрированной ранее команды из журнала КДУ необходимо выбрать команду в окне «Команды диспетчерского управления» ГТП или общего окна «Журнал КДУ» и нажать кнопку «Удалить КДУ» () на панели инструментов.

Для подтверждения операции удаления необходимо нажать кнопку «Да», для отмены – «Нет».

Если кнопка «Удалить КДУ» () недоступна – это означает, что с момента регистрации команды прошло более 24х часов и удаление такой команды запрещено.

Контроль режима генерирующего объекта

Приложение DGViewer предоставляет функции контроля режима по активной мгновенной мощности (МВт), а также по объёмам выработки (потребления) электроэнергии (МВт*ч).

Окна «График оперативный», «Контроль объёмов» и «Оперативные данные» используются для контроля режима работы. Подробное описание этих окон дано в соответствующих разделах настоящего руководства, а описание индикации статусов значений приведено в приложении 1.

Контроль мгновенной активной мощности

Приложение контролирует мгновенное значение активной мощности в заданном коридоре:

ü  для ГТП генерации (ГОУ) коридор имеет границу минимума PУДГ-2% (но не менее PУДГ-5МВт) и границу максимума PУДГ+2% (но не более PУДГ+5МВт) от УДГ.

ü  для ГТП потребления (СН) в приложении реализовано два вида коридоров: предупредительный и аварийный. Предупредительный коридор имеет границы PСН±5%, а аварийный – PСН±10% от графика потребления. В отличие от ГТП генерации, ограничение по предельной величине мощности отсутствует.

Пример 1: если в соответствии с УДГ необходимо держать P равную 100 МВт, то станции необходимо держать режим в следующем коридоре: PMIN=98 МВт, PMAX=102 МВт. В данном случае применяется ограничение PУДГ ±2%.

Пример 2: если в соответствии с УДГ необходимо держать P равную 500 МВт, то станции необходимо держать режим в следующем коридоре: PMIN=495 МВт, PMAX=505 МВт. В данном случае применяется ограничение PУДГ ±5 МВт.

График УДГ строится на основе КДУ на основе ДДГ. Относительно графика потребления (СН) КДУ не выдаются.

При нарушении режима – выходе значения P за границы коридора, отображается соответствующая индикация (см. Приложение 1. Индикация статусов значений). Для привлечения внимания вокруг окна «График оперативный» появляется дополнительная рамка ярко красного цвета.

Дополнительная информация о текущем режиме представлена в окне «Оперативные данные»:

ü  индикатор «Рекомендация» отображает рекомендованное значение мгновенной активной мощности (МВт), которое необходимо поддерживать в текущий момент времени для обеспечения, заданной планом, выработки (потребления) электроэнергии на конец интервала.

ü  индикатор «Прогноз выработки» отображает прогнозируемый объём выработки (потребления) электроэнергии (МВт*ч) на конец интервала, если текущее значение активной мощности будет неизменным.

Значения «Рекомендация» и «Прогноз выработки» рассчитываются в реальном времени с использованием линейных функций c учётом паспортных характеристик оборудования.

Для расчета рекомендации по активной мощности необходимо указать параметр «Максимальная мощность» (в МВт) в паспортных характеристиках оборудования для каждого генератора, входящего в состав ГТП.

Контроль объёмов электроэнергии

Приложение выполняет контроль объёмов выработки (потребления) электроэнергии в двух режимах:

ü  контроль прошедших временных интервалов;

ü  контроль текущего временного интервала.

Контроль прошедших временных интервалов выполняется в соответствии с выбранной датой и интервалом (час/полчаса). При этом рассчитанные значения фактических объёмов электроэнергии сравниваются с плановыми. Также производится вычисление отклонения в абсолютных значениях (МВт*ч) и в процентах. Проверяется выход рассчитанного отклонения за установленные коридоры.

Используется два коридора: предупредительный (V±5%) и аварийный (V±10%). При выходе значения объёма за границы коридора ячейка таблицы раскрашивается в соответствии с приложением 1.

В рамках текущего интервала времени (оперативный час/полчаса) выполняется ежеминутный расчёт фактических объёмов выработки (потребления) электроэнергии и вычисление отклонений от плановых. Значения текущего интервала времени отображаются отдельной строкой в таблице, выделенной жирным шрифтом. Объём электроэнергии является величиной накопительной, поэтому в текущий момент времени контроль отклонений в соответствии с установленными коридорами не производится, а производится по завершению текущего временного интервала (режим контроля прошедших временных интервалов).

Данные коридоры рассчитываются от УДГ для генерации и от графика потребления для СН.

Плановые значения выработки (потребления) электроэнергии вычисляются на основе плановых графиков: УДГ для ГТП генерации (ГОУ) и план потребления для ГТП потребления (СН). Вычисление объёма выполняется с помощью интеграла, рассчитанного по методу трапеций с минутными интервалами.

Приложение 1. Индикация статусов значений

 

Любой отображаемый параметр в программе поддерживает цветовую индикацию статуса значения. Этот статус указывает о возможной проблеме, нештатной работе системы или нарушении режима. В зависимости от статуса, индикация может быть представлена в виде рамки вокруг элемента или в виде цветного фона элемента.

 – отсутствует источник значения; значение в этой ячейке нельзя использовать для контроля режима; возможно, имеются проблемы со связью у сервера БДРВ или неисправен измеритель;

Приложение 2. Конфигурационные файлы

 

Приложение DGViewer хранит пользовательские настройки в конфигурационных файлах, расположенных в каталоге:

C:\Users\%UserName%\AppData\Local\EMA\RSDU\DGViewer

Примечание. Папка «Users» в русифицированной версии операционной системы Windows отображается как «Пользователи», а папка «AppData» является скрытой.

К пользовательским настройкам относятся: все настройки, перечисленные в главе «Настройки приложения», положение и размер окон, настройки визуализации форм отображения журнала КДУ.

В зависимости от параметров запуска DGViewer пользовательские данные могут храниться в конфигурационных файлах с разными именами. Подробности организации конфигурационных файлов представлены в разделе настоящего документа «Приложение 3. Параметры запуска приложения».

Для того чтобы сбросить все настройки программы в значения по умолчанию, необходимо удалить все файлы из этого каталога или непосредственно сам каталог.

Приложение 3. Параметры запуска приложения

 

Приложение «Диспетчерские графики» (DGViewer.exe) можно запустить со следующими параметрами в командной строке:

ü  -s или –stations - список идентификаторов станций через запятую, для которых нужно запустить приложение;

ü  -a или --app_id - идентификатор приложения. Используется для разделения локальных настроек приложения.

Задать входные параметры запуска приложения можно при вызове приложения из командной строки Windows или в настройках приложения с помощью «Навигатора БД» (DBView.exe) в разделе «Служебная информация → Программное обеспечение». Подробная информация о работе с разделом «Программное обеспечение» представлена в руководстве «Управление разделами БД РСДУ», картридж «Приложения РСДУ».

Пример запуска приложения «Диспетчерские графики» с параметрами:

DGViewer.exe -s 2,225,75 -a 10025

Получение идентификаторов станций, описанных в разделе «Диспетчерские графики» производится с помощью приложения «Навигатор БД». Идентификатор выбранного элемента в каталоге раздела отображается в строке состояния приложения, в поле «ID узла», а также в наименовании окна картриджа управления элементами каталога раздела «Диспетчерские графики».
