1 Термины, определения, обозначения и сокращения

В настоящем документе приняты следующие термины, определения, обозначения и сокращения:

API (англ. Application Programming Interface) – интерфейс программирования приложений, интерфейс прикладного программирования – набор готовых классов, процедур, функций, структур и констант, предоставляемых приложением (библиотекой, сервисом) для использования во внешних программных продуктах.

MODES-Centre – программный комплекс, предназначенный для автоматизации бизнес-процессов распределенной генерирующей компании, связанных с хранением и анализом на уровне центра управления информации недельного, суточного и оперативного планирования, обмен которой осуществляется между подразделениями генерирующей компании и информационными шлюзами СО ЕЭС.

ODBC (англ. Open Database Connectivity) – программный интерфейс (API) доступа к базам данных, разработанный фирмой Microsoft.

SQLite – компактная встраиваемая реляционная база данных, используемая для хранения локальных данных.

БДТИ – База Данных Технологической Информации РСДУ.

ГОУ (Групповой объект управления) – совокупность одной или нескольких групп точек поставки (ГТП), в отношении которой Системным оператором отдаются и фиксируются команды на изменение режима работы.

ГТП (Группа точек поставки) – одна или несколько точек поставки, относящихся к одному узлу расчетной модели и/или к единому технологически неделимому энергетическому объекту и ограничивающих территорию, в отношении которой купля-продажа электрической энергии (мощности) на оптовом рынке осуществляется только данным участником, и используемых для определения и исполнения участником оптового рынка обязательств, связанных с поставкой и оплатой электрической энергии (мощности). Группа точек поставки определяется Системным и Коммерческим операторами в соответствии с Положением о порядке получения статуса субъекта оптового рынка и ведения реестра субъектов оптового рынка электрической энергии и мощности.

Диспетчерский график, Плановый график – задает план нагрузки (значения активной мощности на границе интервалов) для ГОУ и РГЕ расчетной модели.

КДУ – Команда диспетчерского управления.

ОИК – Оперативный Информационный Комплекс.

РГЕ – Группа единиц генерирующего оборудования, относящихся к одной группе точек поставки и привязанных к одному узлу расчетной модели. Каждая режимная генерирующая единица имеет свой идентификационный код в расчетной модели.

СО (СО ЕЭС) – Системный Оператор Единой Энергосистемы Российской Федерации.

 

2 Общие сведения

Название сервиса: RSDU.INP\IAModesCentre.

2.1 Назначение приложения

Сервис получения диспетчерских графиков из MODES-Centre предназначен для получения плановых диспетчерских графиков (ДГ) из внешней системы MODES-Centre и записи этих данных в архивы соответствующих параметров раздела «Диспетчерские графики» БДТИ РСДУ5.

2.2 Необходимые условия

Перед переходом к действиям, описанным ниже в руководстве, должны быть выполнены следующие необходимые условия:

ü  Установлена ОС Microsoft Windows 2008 или выше;

ü  Установлен пакет .NET Framework 4.5;

ü  Установлен пакет Visual C++ 2010 Redistributable Package;

ü  Установлен пакет клиента Oracle 11.2.0.2;

ü  Настроен источник ODBC на БДТИ РСДУ;

ü  Выполнены обновления в БДТИ РСДУ:

o   2013_12_02_DG_NEW;

o   2014_03_04_J_DGSTATE.

2.3 Необходимые знания

Для выполнения действий, описанных ниже в руководстве, требуются следующие знания:

ü  Имя пользователя ОС, имеющего права администратора, и его пароль;

ü  Имя пользователя РСДУ, от имени которого будет запускаться сервис переноса графиков;

ü  Адрес сервера MODES-Centre;

ü  Учетная запись (имя пользователя и пароль) MODES-Centre с разрешенными правами на чтение плановых графиков и информационной модели;

ü  Адрес сервиса доступа к архивам arch-webapi.

2.4 Ссылки

В руководстве используются ссылки на следующую документацию к комплексу РСДУ:

ü  «Описание серверов и точек доступа в комплексе РСДУ. Руководство администратора».

4 Установка сервиса

Необходимо войти в операционную систему Windows пользователем с правами администратора.

1.    Скопировать дистрибутив сервиса (каталог RSDU.INP\IAModesCentre) на целевой хост в каталог C:\RsduServices\RSDU.INP;

2.    Выполнить установку сервиса с помощью скрипта 01. Install.cmd.

Для этого необходимо открыть командную строку Windows от имени Администратора. Далее следует перейти в каталог размещения сервиса и выполнить скрипт 01. Install.cmd.

Рисунок 3 – Пример вывода скрипта 01. Install.cmd

После выполнения скрипта 01. Install.cmd сервис должен отображаться в консоли управления Службами Windows (Services).

Название сервиса Windows следующее: RSDU.INP> Интерфейсный адаптер чтения графиков из MODES-Centre.

3.    Выполнить конфигурирование сервиса с помощью приложения IAModesCentre.Config.exe (см. раздел настоящего руководства «Конфигурирование сервиса»);

4.    Выполнить настройку автоматического запуска и восстановления сервиса.

Для настройки автоматического запуска сервиса необходимо открыть свойства сервиса RSDU.INP> Интерфейсный адаптер чтения графиков из MODES-Centre в консоли управления Службами Windows (Services), перейти во вкладку «Общие» и указать тип запуска «Автоматически».

Настройка параметров восстановления при сбоях выполняется во вкладке «Восстановление». При этом необходимо указать для Первого, Второго и Последующих сбоев перезапуск через 1 минуту.

6 Конфигурирование сервиса

Для настройки сервиса используется приложение IAModesCentre.Config.exe, расположенное в каталоге установки сервиса C:\RsduServices\RSDU.INP\IAModesCentre.

Рисунок 5 – Интерфейс приложения настройки сервиса вкладка «Настройки»

Окно конфигуратора состоит из следующих вкладок:

ü  Настройки – общие настройки сервиса, такие как параметры подключения к системам, периоды опроса, часовые пояса и т.д.;

ü  Связь справочников – содержит описание элементов справочников интегрируемых систем и связей между ними;

ü  Рабочие переменные – это переменные, которые сохраняются при остановке сервиса и используются при его старте. Рабочие переменные могут быть отредактированы.

Конфигурация сервиса хранится в файлах БД SQLite в каталоге db:

ü  config.db – файл БД конфигурации сервиса. Содержит общие настройки сервиса (вкладка «Настройка») и описание справочников интегрируемых систем (вкладка «Связь справочников»);

ü  work.db – файл БД для хранения рабочих переменных.

Для удобства выполнения необходимо дополнительно запустить приложение «Навигатор БД» комплекса РСДУ (запустить Панель оператора и вызвать приложение Навигатор БД).

6.2.2 Справочник ГОУ/РГЕ

Справочник ГОУ/РГЕ содержит описание генерирующей компании, филиалов, станций, групповых объектов управления, плановых графиков и т.д.

Соответствующие элементы справочника должны содержать идентификаторы аналогичных элементов в системах MODES-Centre и РСДУ.

6.2.2.1 Описание структуры справочника

Справочник оперирует следующими типами элементов:

ü  Объект модели (OBJECT_MODEL) – тип определяет любой объект модели в иерархии;

ü  ГОУ (OBJECT_GOU) – тип определяет элемент ГОУ (групповой объект управления);

ü  РГЕ (OBJECT_RGE) – тип определяет элемент РГЕ (режимную генерирующую единицу);

ü  Плановый график (PDG) – тип определяет плановый диспетчерский графики;

Элементы с типами ГОУ (или РГЕ) и Плановый график должны быть связаны, то есть должны быть указаны идентификаторы соответствующих элементов справочников MODES-Centre и РСДУ.

6.2.2.2 Заполнение справочника

Приложение настройки сервиса поддерживает автоматизированное заполнение справочника ГОУ/РГЕ на базе справочников MODES-Center.

При этом приложение выполняет подключение к серверу MODES-Centre (используя параметры подключения, заданные во вкладке «Настройки»), производит запрос информационной модели MODES-Centre и на базе полученных данных заполняет «Справочник ГОУ/РГЕ».

Дальнейшая привязка (указание идентификаторов РСДУ) элементов сформированного справочника производится вручную.

2.    Запустить Панель оператора и приложение «Навигатор БД» РСДУ.

Приложение «Навигатор БД» потребуется на следующих шагах настройки для поиска идентификаторов элементов раздела «Диспетчерские графики» в процессе привязки справочника;

3.    Нажать кнопку Загрузить модель ().

Приложение выполнит загрузку всей текущей (на момент запроса) модели с сервера MODES-Centre и отобразит сформированный справочник.

2.    Из командной строки Windows.

Для работы скриптов необходимы права администратора.

1.    Открыть командную строку Windows с правами администратора.

2.    Перейти в каталог установки сервиса.

По умолчанию каталог установки:

C:\RsduServices\RSDU.INP\IAModesCentre

3.    Запустить скрипт 02. Start.cmd.

7.1.2 Запуск/останов из консоли управления службами Windows

Для запуска или остановки сервиса с использованием консоли управления службами Windows необходимо выполнить следующие операции:

1.    Открыть консоль управления службами Windows: «Пуск → Панель управления → Администрирование → Службы»

2.    В списке служб найти и выделить службу RSDU.INP> Интерфейсный адаптер чтения графиков из MODES-Centre.

3.    Для запуска службы нажмите кнопку «Запуск службы».

4.    Для остановки  службы нажмите кнопку «Остановка службы».

Текущее состояние службы отображается в колонке «Состояние» в списке служб.

Работу сервиса можно проконтролировать по сообщениям в лог-файле.

7.2 Управление рабочими переменными

Рабочие переменные сервиса – это переменные, которые сервис сохраняет в файл в темпе их изменения в процессе работы и читает из файла при запуске.

внимание!

Редактирование переменных возможно только при остановленном сервисе.

Для изменения значения переменной выберите ячейку колонки «Значение», введите новое значение и нажмите Enter или переместите фокус на другой элемент или строку.

Кнопка «Обновить» производит обновление значений переменных из файла.

7.3 Контроль лог-файлов

Лог-файл сервиса располагается в каталоге установки сервиса и называется IAModesCentre.Service.log.

Например,

C:\RsduServices\RSDU.INP\IAModesCentre\IAModesCentre.Service.log

Раз в неделю сервис выполняет архивирование и очистку лог-файла. Архивный лог-файл сжимается и копируется в каталог archivelogs. В имени архивированного лог-файла будет содержаться дата архивации.

Например,

C:\RsduServices\RSDU.INP\IAModesCentre\archivelogs\ IAModesCentre.Service_2016-01-07.log.gz

7.4 Сигнализация

Сервис переноса диспетчерских графиков из MODES-Centre в процессе работы отправляет в сигнальную систему РСДУ следующие сигналы:

ü  Загрузка и инициализация (SS_APLOADINIT) – формируется при успешном запуске сервиса;

ü  Завершение приложения  (SS_NORMALQUIT) – формируется при остановке сервиса;

ü  Получен диспетчерский график  (SS_DGLOAD) – сигнал информирует о приеме нового планового графика;

ü  Изменен диспетчерский график  (SS_DGCHANGE) – формируется при обновлении значений ранее принятого диспетчерского графика, например, при получении ПБР;

ü  Ошибка при получении диспетчерского графика (SS_DGFAIL) – формируется в случае, если в процессе получения и записи в архивы возникла ошибка. Более подробная об ошибках информация при этом содержится в лог-файле сервиса.
