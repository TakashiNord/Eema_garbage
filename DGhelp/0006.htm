Общие сведения

 

Назначение сервиса

Сервис предназначен для переноса диспетчерских графиков (ДГ) из внешней системы Модес-Т в разделы суточных графиков и диспетчерских графиков БДТИ РСДУ5. Сервис также служит для переноса диспетчерских графиков и команд диспетчерского управления (КДУ) в удалённую базу данных комплекса РСДУ5 (например, БДТИ РСДУ5 центрального офиса компании).

Необходимые условия

ОС Microsoft Windows 2000 или выше.

Установлен пакет .NET Framework 4.0.

Установлен пакет Visual C++ 2005 Redistributable Package.

Установлен пакет клиента Oracle 11.2.

Настроен источник ODBC на БДТИ РСДУ.

Выполнены обновления в БДТИ РСДУ «2013_12_02_DG_NEW» и «2014_03_04_J_DGSTATE».

Дистрибутив «DgFromModesTService».

Необходимые знания

Имя пользователя ОС, имеющего права администратора и его пароль.

Имя пользователя, от имени которого будет запускаться сервис переноса графиков.

Строка соединения с БД Модес-Т.

Установка сервиса

 

Необходимо войти в операционную систему Windows под пользователем с правами администратора.

1.     Скопировать папку с дистрибутивом сервиса DgFromModesT на целевой хост в папку C:\RsduServices\;

2.     При необходимости отредактировать инсталляционные командные файлы Install.cmd, Start.cmd, Stop.cmd, Uninstall.cmd;

3.     Выполнить скрипт по установке сервиса Install.cmd;

4.     Выполнить настройку сервиса с помощью приложения SettingsLauncher.exe (см. раздел настоящей инструкции «Конфигурирование сервиса»);

5.     В случае необходимости вывода расширенной отладочной информации необходимо в конфигурационном файле «ServiceSettings.xml» следует указать:

ü При необходимости вывода отладочной информации поставить флаги в разделах <CommonDebug> и <DbDebug> в true;

ü В разделе <TraceWriterSettings> настроить параметры:

a.  <DayLimit> - количество дней для записи лога, прежде чем файл лога сменится на новый, а предыдущий отправится в архив;

b.  <SizeLimitMb> - предел на размер файла лога в Мб, если не сработал <DayLimit>;

c.   <DayCyrcle> - количество дней, которое хранятся архивы логов;

d.  <SizeCyrcleMb> - предел на размер архивов логов если не сработал <DayCyrcle>.

6.     Открыть список служб в настройках ОС Windows («Панель инструментов/ Администрирование/Службы»);

7.     Найти сервис «Import Disp Graphics From Modes-T», указать тип запуска «Авто»; выбрать вкладку «Восстановление» и указать для Первого, Второго и т.д. сбоев перезапуск через 1 минуту;

8.     Для ручного запуска сервиса необходимо в контекстном меню выбрать команду «Пуск», для останова – команду «Стоп».

Настройка параметров связи с Модес-Т

Группа настроек параметров связи с Модес-Т определяет метод работы  и период опроса данных Модес-Т.

Получение данных из системы Модес-Т реализовано несколькими способами:

ü DLL – использование API Модес-Т;

ü DB – использование непосредственной работы с БД Модес-Т.

Метод DLL рекомендуется указывать при настройке сервиса в неценовых зонах. Во всех остальных случаях рекомендуется  использовать метод DB.

Дальнейшая настройка параметров соединения с Modes-T не зависит от выбранного метода.

Параметр «Период опроса данных (мин)» определят период (в минутах) отправки в систему Модес-Т запросов на получение информации по обрабатываемым графикам. Рекомендуемые значения – 10, 15 или 30 минут.

Настройка параметров соединения с Модес-Т

Чтобы приступить к настройке переноса данных из Модес-Т нажмите кнопку «Создать», расположенную под списком станций, слева внизу.

Рисунок 6 - Добавление нового соединения с Модес-Т

Далее необходимо ввести строку соединения в поле «Строка соединения».

Например:

При этом текст строки в списке станций изменится на значение параметра Initial Catalog из строки соединения.

Администрирование сервиса

 

Запуск сервиса осуществляется автоматически вместе со стартом операционной системы. В сигнальную систему должно приходить сообщение о загрузке сервиса. Если в сигнальной системе сообщения нет или присутствуют сообщения об ошибках в работе сервиса, то для получения дополнительной информации следует обратиться к файлу лога DgFromModesTService.log. Файл лога расположен в той же папке что и сервис.

Когда сервис получает новые данные из Модес-Т, он отправляет в сигнальную систему сообщение о получении нового графика. Если при этом есть необходимость переинициализации сервера суточных графиков, сервис вызывает утилиту HGControl.exe. Утилита посылает команду переинициализации, а результат команды записывается в системный журнал приложений ОС Windows.
