1 Общие сведения

1.1 Назначение

Документ содержит описание по установке и настройке системы управления контейнерами Docker.

Docker – это программное обеспечение для развёртывания и управления приложениями в виде автономных контейнеров[1]. Позволяет запускать множество контейнеров на одной хост-машине.

Контейнеры — это способ упаковать приложение и все его зависимости в единый образ. Этот образ запускается в изолированной среде, не влияющей на основную операционную систему. Контейнеры позволяют отделить приложение от инфраструктуры. В контейнере может быть запущен экземпляр операционной системы только с тем же ядром, что и у хостовой операционной системы (все контейнеры узла используют общее ядро).

1.2 Необходимые условия

Перед переходом к действиям, описанным ниже в руководстве, должны быть выполнены следующие необходимые условия:

ü  установлена операционная система[2] Debian 10 или Debian 11;

ü  наличие дистрибутива Docker для выбранной ОС в составе containerd.io_<version>.deb, docker-ce-cli_<version>.deb, docker-ce_<version>.deb последних версий:

o   Debian 10:

https://download.docker.com/linux/debian/dists/buster/pool/stable/amd64/

o   Debian 11: https://download.docker.com/linux/debian/dists/bullseye/pool/stable/amd64/

ü  наличие дистрибутива Docker Compose для выбранной ОС в составе docker-compose-Linux-x86_64:

o   Debian 10 и Debian 11:

https://github.com/docker/compose/releases

ü  наличие дистрибутива панели управления Portainer в составе portainer-ce-docker-<version>.tar.gz

 

 

1.3 Необходимые знания

Для выполнения действий, описанных ниже в руководстве, требуются следующие знания:

ü  навыки администрирования ОС Linux;

ü  навыки администрирования Docker;

ü  доступ в ОС Linux с правами root.

навыки работы с интернет браузером.

2 Установка и настройка окружения

2.1 Установка Docker

Для установки Docker необходимо выполнить следующее:

1.    Подключиться с правами суперпользователя root.

2.    Удалить старые версии.

# apt-get remove docker docker-engine docker.io containerd runc

3.    Установить необходимые пакеты:

# apt-get install libip6tc? \

                libnfnetlink0 \

                libnetfilter-conntrack3 \

                iptables

4.    Скопировать на хост дистрибутив Docker (containerd.io_<version>.deb, docker-ce-cli_<version>.deb, docker-ce_<version>.deb).

5.    Перейти в каталог с дистрибутивом.

6.    Установить Docker[3]:

# dpkg -i containerd.io_<version>.deb
# dpkg -i docker-ce-cli_<version>.deb
# dpkg -i docker-ce_<version>.deb

7.    Убедиться, что Docker установился:

# docker –-version

Docker version 20.10.10, build b485636

 

2.2 Установка Docker Compose

Docker Compose – программное обеспечение, позволяющее развернуть и настроить приложения Docker по заранее составленной инструкции (docker-compose.yml)[4].

Для установки Docker Compose необходимо:

1.    Скопировать на хост файл docker-compose-Linux-x86_64 из дистрибутива Docker Compose.

2.    Выполнить команду копирования файла в каталог /usr/bin/docker-compose.

# cp -f docker-compose-Linux-x86_64 /usr/bin/docker-compose

3.    Дать файлу права на исполнение:

# chmod +x /usr/bin/docker-compose

4.    Убедиться, что Docker Compose установлен:

# docker-compose version

docker-compose version 1.29.2, build 5becea4c

2.3 Настройка Docker

Для настройки Docker необходимо выполнить следующее (если работа планируется не под root):

1.    Добавить права пользователям хоста на команды запуска Docker (по необходимости, в примере ниже использован пользователь admin).

# usermod -aG docker admin

2.    Для того, чтобы изменения вступили в силу, нужно выйти и снова залогиниться на хост под пользователем (в нашем случае admin). После этого посмотреть, в каких группах состоит пользователь. Должна появиться группа 998(docker).

# id $admin

uid=1001(admin) gid=1001(admin) группы=1001(admin),998(docker)

2.4 Установка панели управления Portainer

Для более удобного управления Docker предлагается использовать панель управления Portainer.

Для установки необходимо выполнить следующие команды:

1.    Скопировать на хост дистрибутив панели управления Portainer в составе portainer-ce-docker-<version>.tar.gz.

2.    Перейти в каталог с дистрибутивом.

3.    Импортировать образ Portainer (в примере команды ниже используется версия 2.9.2):

# docker load --input portainer-ce-docker-2.9.2.tar.gz

4.    Добавить виртуальное пространство под данные:

# docker volume create portainer_data

5.    Инициализировать и запустить Portainer (в примере команды ниже используется версия 2.9.2):

# docker run -d -p 8000:8000 -p 9000:9000 \

  --name=portainer-ce \

  --restart=always \

  -v /var/run/docker.sock:/var/run/docker.sock \

  -v portainer_data:/data \

  portainer/portainer-ce:2.9.2

После установки панель управления доступна на порту 9000.

Примечание. Если в процессе работы, контейнер с панелью управления Portainer был случайно удалён, необходимо повторно выполнить команду docker run, которая указана выше.

6.    Для проверки работы панели управления открыть на АРМ в браузере адрес <IP-адрес хоста>:9000.
