1 Термины, определения, обозначения и сокращения

В настоящем документе приняты следующие термины, определения, обозначения и сокращения:

systemd – стандартная система инициализации, запуска и управления фоновыми процессами в ОС Linux.

БДТИ – База Данных Технологической Информации РСДУ.

Лог-файл – текстовый файл с записями о событиях, происходящих в программном обеспечении в хронологическом порядке.

Пакет обновления БД – набор скриптов, предназначенных для внесения изменений в структуру и данные таблиц БДТИ комплекса.

 

 

2 Общие сведения

2.1 Назначение сервиса

Сервер приложений SCADA (далее Сервер SCADA или Сервис) представляет собой серверный программный модуль, обеспечивающий высокопроизводительный доступ к функциям комплекса для клиентских приложений.

Высокая производительность Сервера SCADA достигается за счет кэширования модели и данных при выполнении функций.

С целью резервирования в комплексе может быть установлено несколько Серверов SCADA, при этом только один экземпляр работает в режиме основной (master). Некоторые функции сервера выполняются только в режиме основной.

Поддерживается работа SCADA сервера в операционной системе Linux.

2.2 Необходимые условия

ОС Linux:

ü  Ubuntu 16.04 или выше;

ü  Debian 9 или выше;

ü  openSUSE 15 или выше;

ü  SUSE Enterprise Linux (SLES) 12 SP2 или выше.

Установлен прокси-сервер Nginx версии 1.14.2 или выше.

Установлен сервер авторизации РСДУ.

Установлены следующие пакеты обновления БДТИ:

ü  2020_02_10_JSWITCHES_APP.

2.3 Необходимые знания

Дистрибутив серверной части РСДУ:

RSDU5_SCADA_linux-x64-<version>-YYYYMMDD_HHMM-<revision>.tar.bz2

Название сервиса: scada-server.

Для запуска сервиса необходимо знать:

ü  имя пользователя ОС, имеющего права администратора, и его пароль на сервере установки сервиса;

ü  имя пользователя и пароль учетной записи РСДУ, используемой сервисом для подключения к РСДУ;

ü  имя пользователя и пароль учетной записи администратора РСДУ, для выполнения операций настройки сервиса;

ü  параметры подключения к БДТИ и другим системным ресурсам комплекса РСДУ.

2.4 Ссылки

Для настройки сервиса в БДТИ РСДУ ознакомьтесь со следующими документами:

ü  «Описание оборудования и точек доступа в комплексе РСДУ5. Руководство администратора»;

ü  «Структура базы данных. Описание структуры БД РСДУ. Руководство администратора»;

3 Создание учетной записи РСДУ

Для работы Сервера SCADA требуется учетная запись РСДУ. Сервер использует эту учетную запись для подключения к БД и другим ресурсам комплекса.

Пакет обновления 2020_02_10_JSWITCHES_APP выполняет регистрацию исполняемого модуля (приложения/файла) сервера SCADA в разделе «Служебная информация → Программное обеспечение → Программы → Загружаемые модули».

Для создания учетной записи РСДУ необходимо:

1.    Запустить приложение «Навигатор БД» РСДУ и перейти в раздел «Доступ к информации → Процессы и персонал».

2.    В ветке «Процессы» раздела определить группу, в которой будет создана учетная запись для Сервера SCADA. Как правило, группы в ветке «Процесс» организованы по хостам комплекса.

Выбрать необходимую группу и вызвать пункт «Создать» контекстного меню области списка Навигатора БД.

В открывшемся окне указать соответствующее приложение, имя и логин учетной записи. Выбор приложения осуществляется по кнопке «Выбрать» из приложения «Настройка Базы Данных – Adjust».

В поле «Тип» должно быть задано «Процесс», в поле «Подсистема» – «Базовая информация комплекса».

Рисунок 1 – Окно создания учетной записи РСДУ с типом «Процесс»

3.    Завершение создания учетной записи выполнить кнопкой «Готово».

Права доступа назначаются автоматически при создании учетной записи на основе описания приложения «Сервер приложений SCADA».

По умолчанию для учетных записей с типом «Процесс» задается пароль «rsdu_211».

 

4 Установка и управление сервером SCADA

4.1 Установка сервиса

Установка сервиса проводится от имени пользователя с правами root. Если в используемой ОС Linux заблокирован вход от имени пользователя root, то необходимо использовать команду sudo для выполнения операций.

1.    Скопировать архив с дистрибутивом Сервера SCADA для ОС Linux на целевой хост в домашний каталог пользователя.

Имя файла дистрибутива: RSDU5_SCADA_linux-x64-<version>-YYYYMMDD-HHMM-<revision>.tar.bz2

2.    Выполнить распаковку архива в домашний каталог пользователя

tar -xjf RSDU5_SCADA_linux-x64-<version>-YYYYMMDD-HHMM-<revision>.tar.bz2

3.    Выполнить скрипт установки сервера install.sh.

cd ~/linux-x64/scada-server/Installers/Linux/

./install.sh

Рабочий каталог сервера: /usr/lib/ema/rsdu/scada-server

Каталог конфигурационных файлов: /etc/ema/

4.    Скопировать конфигурационные файлы для настройки.

В директории ~/linux-x64/scada-server/Installers/Linux находится шаблон файла конфигурации scada-server.default.json.

Необходимо скопировать шаблон в директорию /etc/ema под именем scada-server.json.

cp scada-server.defaults.json /etc/ema/scada-server.json

5.    Выполнить редактирование скопированного конфигурационного файла, согласно разделу «Конфигурирование сервера SCADA».

6.    При необходимости удалить дистрибутив:

rm –R ~/linux-x64

rm ~/RSDU5_SCADA_linux-x64-<version>-YYYYMMDD-HHMM-<revision>.tar.bz2

4.2 Запуск и останов сервиса

Управление работой Сервера SCADA в ОС Linux осуществляется через стандартные команды systemd. Имя сервиса в systemd: rsdu-scada-server.

Запуск сервиса:

systemctl start rsdu-scada-server

Останов сервиса:

systemctl stop rsdu-scada-server

Перезапуск сервиса:

systemctl restart rsdu-scada-server

Контроль текущего состояния:

systemctl status rsdu-scada-server

4.3 Контроль лог файлов

Лог-файлы в ОС Linux Сервера SCADA располагаются в каталоге:

/var/log/ema/scada-server

Сервер ведет следующие лог-файлы:

ü  scada-server.log – рабочий лог-файл. Хранение до месяца;

ü  scada-server_debug.log – отладочный лог. Хранение несколько дней;

ü  scada-server_web.log – лог работы WEB API. Хранение несколько дней.

Примечание. Если после команды запуска сервера лог-файлы не появились, ошибки запуска следует смотреть в системном журнале ОС, используя команду journalctl -u rsdu-scada-server.

4.4 Удаление сервиса

Удаление сервиса выполняется от имени пользователя с правами root при помощи скрипта uninstall.sh

cd /usr/lib/ema/rsdu/scada-server

./uninstall.sh

Примечание. Скрипт uninstall.sh не удаляет конфигурационные файлы.

5.1 Общая конфигурация

Urls – IP-адрес и порт сервиса.

HostStatusInformerOptions / DefaultHostStatus – статус сервиса при запуске. Доступные варианты: Slave (резервный), Master (основной).

Если предполагается запускать только один экземпляр сервиса без резервирования (односерверная конфигурация), необходимо указать Master. В случае резервирования каждому экземпляру в зависимости от предполагаемой по умолчанию роли, при этом экземпляр со статусом Master должен быть один.

Блок HostStatusFromFileOptions – параметры модуля определения текущего статуса сервиса.

ü  Enabled – true (модуль включён). Для односерверной конфигурации модуль необходимо отключить, указав false;

ü  FilePath – путь к файлу с текущим статусом сервиса.

Блок DataBase – параметры подключения к БД.

ü  DataSource – строка с информацией о подключении к БД в формате «IP_адрес:порт/имя_бд». Вместо IP_адреса может использоваться DNS имя;

ü  Login – логин учетной записи;

ü  Password – пароль учетной записи;

ü  Application – alias зарегистрированного сервера в разделе Навигатора БД РСДУ «Служебная информация → Программное обеспечение → Программы → Загружаемые модули» (по умолчанию «scada-server»).

Блок RabbitMQ – параметры подключения к брокеру сообщений Rabbit.

ü  Login – логин учетной записи;

ü  Password – пароль учетной записи;

ü  VirtualHost – виртуальный хост для обмена сообщениями, используемый в комплексе. По умолчанию – rsdu;

ü  Port – порт подключения. По умолчанию – 5672;

ü  HostNames – список серверов RabbitMQ. Задаются в виде ip-адресов или имен DNS.

Блок UserServiceOptions – параметры подключения к серверу авторизации.

ü  AuthServerUrl – Url подключения к серверу авторизации.

Блок ArchivesServiceOptions / OracleModelDB – настройки БД Oracle для доступа к БДТИ с конфигурацией комплекса РСДУ – используется модулем чтения и записи архивов.

ü  DataSource – строка с информацией о подключении к БД Oracle в формате «IP_адрес:порт/имя_бд». Вместо IP_адреса может использоваться DNS имя;

ü  Login – логин учетной записи;

ü  Password – пароль учетной записи;

ü  UseOracleArchives – использовать БД Oracle для хранения архивов;

ü  UseCassandraArchives – использовать БД Cassandra для хранения архивов.

Блок ArchivesServiceOptions / OracleArchivesDB – настройки БД Oracle для хранения архивов.

ü  DataSource – строка с информацией о подключении к БД Oracle в формате «IP_адрес:порт/имя_бд». Вместо IP_адреса может использоваться DNS имя;

ü  Login – логин учетной записи;

ü  Password – пароль учетной записи

Блок ArchivesServiceOptions / CassandraArchivesDB – настройки БД Cassandra для хранения архивов.

ü  Hosts – список IP-адресов или DNS имён;

ü  Port – порт;

ü  Username – имя пользователя;

ü  Password – пароль пользователя;

ü  Keyspace – имя пространства хранения данных.

Блок Logging – конфигурация логирования. Не должна изменяться пользователем.

Дополнительные настройки вывода сообщений в лог так же содержатся в файле /usr/lib/ema/rsdu/scada-server/appsettings.json.

5.2 Конфигурация модулей приложений

5.2.1 Приложение просмотра схем

Блок TopologyServerOptions – конфигурация подключения к серверу топологии.

ü  ProcessIds – список идентификаторов пользователей серверов топологии в разделе Навигатора БД РСДУ «Доступ к информации и управлению → Процессы и персонал → Процессы».

ü  OicConnectionOptions – опции подключения

o   UseMasterSlave – использовать механизм определения статуса хоста (Master/Slave),

o   ConnectTo – определяет к какому серверу подключаться: 0 – со статусом основной, 1 – со статусом резервный, 2 – к любому.

o   UserId – идентификатор пользователя сервера приложений scada (раздел Навигатора БД РСДУ «Доступ к информации и управлению → Процессы и персонал → Процессы»), логин которого используется при подключении к БД (блок DataBase). От имени этого пользователя будет инициировано подключение, этот идентификатор пользователя будет использоваться в протоколе ADCP.

o   SocketTimeout – таймаут ожидания ответа от сервера в секундах. Необязательный параметр. По умолчанию 20 секунд.

o   ConnectRetryInterval – интервал времени между попытками восстановления соединения с сервером топологии в секундах. По умолчанию 30 секунд.

Блок SchemesOptions – конфигурирование работы модуля.

ü  UseDataEmulation – подключение режима эмуляции данных без подключения к серверу топологии. Используется для отладки. В обычном режиме должно иметь значение false;

ü  NetworkStateService/ControlPeriod – период опроса состояния сети (сек);

ü  TracingService/ControlPeriod – период опроса при трассировке сети (сек);

ü  TracingService/NotAciveLivePeriod – период неактивности задачи трассировки до ее завершения (сек).

Блок VSCommonSettings – общие настройки модуля схем. Содержит настройки цветов, используемых приложением «Просмотр схем» при трассировке и отображении состояния электрической сети.

Блок VSCommonSettings / ENetworkStateSettings содержит список цветовых настроек для отображения состояния сети.

ü  State – состояние, для которого задается цвет;

ü  Name – наименование состояния;

ü  Color – цвет в шестнадцатеричном представлении формата RGB.

Блок VSCommonSettings / ENetworkTracingSettings – список цветовых настроек для отображения трассировки.

ü  TraceType – тип трассировки;

ü  Name – наименование типа трассировки;

ü  Color – цвет в шестнадцатеричном представлении формата RGB

5.2.2 Модуль формирования топологической модели

Блок TopologyServiceOptions/OMSConnection – настройки подключения к серверу ОМС для получения информации о потребителях.

ü  Address – адрес подключения в формате <host>:<port>;

ü  Login – логин учетной записи ОМС;

ü  Password – пароль учетной записи ОМС.

5.2.3 Приложение программ переключений

Блок JobOptions.

ü  TemplateFilePath – расположение шаблона для формирования бланков переключений.

ü  SFormUrl – URL-адрес сервиса «Программы переключений».

5.2.4 Модуль приложения «Журнал отключений»

Блок JSwitch – настройки работы модуля журнала отключений.

ü  ReportTemplatePath – путь к шаблону ведомости отключений.

ü  OutOfWorkReportTemplatePath – путь к шаблону ведомости коммутационных аппаратов с выведенными из работы источниками состояний.

ü  APTSSignals – список сигналов, принимаемых за срабатывание АПТС. Список содержит поле DEFINE_ALIAS сигналов.

ü  APTSTypes – список типов параметров, относящихся к АПТС. Список содержит список DEFINE_ALIAS типов параметров.

5.2.5 Модуль поддержки видеостены

Работает в связке с сервисом управления видеостеной. Поддерживается работа с несколькими видеостенами.

Блок VideowallOptions/Videowalls – настройки подключения. Для каждого сервиса управления видеостеной задается:

ü  Name – наименование;

ü  Url – Url подключения.

5.2.6 Модуль работы с сервисом интеграции с АС Энергия

В комплексе может функционировать несколько экземпляров сервисов интеграции с АС Энергия, работающих в режиме резервирования.

Блок ASEnergy.

ü  AccessToken – токен доступа к сервису. Должен совпадать с настройками сервиса интеграции;

ü  ConnectTimeoutSec – таймаут подключения к сервису интеграции, в секундах;

ü  AccessPoints – настройки подключения для каждого сервиса интеграции

o    Id – идентификатор может быть любой, при этом уникальным в списке,

o    ServerHost – ip-адрес или DNS имя сервера,

o   ServerPort – порт подключения.

6 Конфигурирование NGINX

Для конфигурации доступа к серверу SCADA необходимо прописать его в конфигурации NGINX.

Конфигурация NGINX находится по пути /etc/nginx.

Основные настройки прокси-сервера находятся в файле nginx.conf. Конфигурация сайтов подключаются директивой include и находится в поддиректории conf.d.

Как правило, конфигурация доступа к сервисам РСДУ описана в файле rsdu_<port>.conf и отдельно в файлах *.loc поддиректории rsdu_locations.

Для работы сервера необходимо перейти в директорию /etc/nginx/conf.d/rsdu_locations и добавить scada.loc.

Файл scada.loc

location ~/rsdu/scada/api

    {

       if ($request_method = OPTIONS )

       {

          add_header 'Access-Control-Allow-Origin' '*';

          add_header 'Access-Control-Allow-Methods' 'GET, OPTIONS, HEAD, PUT, DELETE';

          add_header 'Access-Control-Allow-Headers' 'Content-Type';

 

          add_header Content-Length 0;

          add_header Content-Type text/plain;

 

          return 200;

       }

 

       add_header 'Access-Control-Allow-Origin' '*';

       add_header 'Access-Control-Allow-Methods' 'GET, OPTIONS, HEAD, PUT, DELETE';

       add_header 'Access-Control-Allow-Headers' 'Content-Type';

 

       proxy_pass http://localhost:5001;

 

       proxy_http_version 1.1;

       proxy_set_header Upgrade $http_upgrade;

       proxy_set_header Connection $connection_upgrade;

       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

    }

 

7 Настройка клиентского ПО

7.1 Настройка подключения к серверу SCADA

Настройка параметров подключения к серверу SCADA клиентских приложений комплекса РСДУ для ОС Windows находится в реестре Windows.

Адрес сервера SCADA определяется в параметре Url раздела:

HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\EMA\RSDU 2.1.1

Формат адреса: <host>:<port>

В случае двухсерверной конфигурации необходимо указать общий адрес кластера.
