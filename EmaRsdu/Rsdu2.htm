Цветовое определение:

ü желтый цвет рамки - значение параметра недостоверно;

ü зеленый цвет рамки - выбранный источник значений для данного параметра недоступен;

ü ярко-красный цвет рамки - значение параметра находится за пределами аварийных уставок;

ü темно-красный цвет рамки - значение параметра находится за пределами уставок другого уровня (неаварийного);

ü синий квадрат - текущим источником значений для данного параметра является ручной ввод (оператор);

ü оранжевый квадрат - свидетельствует о замещении источника данных системы сбора;

ü красный квадрат - данный параметр настроен на передачу в смежные системы, передача нарушена;

ü красная мигающая рамка - выведенный на элемент аналоговый параметр принял значение ноль;

ü   синяя мигающая рамка - выведенный на элемент аналоговый параметр имел значение ноль и затем принял ненулевое значение.

 

 

Бежевый цвет отлично сочетается с голубым, изумрудным, коричневым, красным, а также с черным цветами.

Белый сочетается со всем. Наилучшее сочетание с красным, синим и черным цветами.

Бирюзовый цвет будет замечательно смотреться с вишнево-красным, кремовым, желтым, коричневым и темно-фиолетовым.

Бледно-желтый - с серым, оттенками красного, синим, а также пурпурным.

Вишневый - с серым, светло-оранжевым, бледно-желтым, бежевым, песчаным.

Голубой цвет даст красивое сочетание с серым, коричневым, красным, розовым и белым.

Желтый хорошо смотрится с лиловым, синим, фиолетовым, серым, черным.

Зеленый - с оранжевым, желтым, коричневым, серым, кремовым, черным и сливочно-белым цветами.

Красный подходит к желтым, белым, бурым, зеленым, синим и черным.

Коричневый цвет сочетается с ярко-голубым, розовым, кремовым, зеленым и бежевым.

Лиловый - с оранжевым, розовым, темно-фиолетовым, оливковым, серым, желтым и белым.

Лимонно-желтый - с вишнево-красным, коричневым, серым, синим.

Малиновый цвет даст хорошее сочетание с белым, черным.

Оранжевый отлично гармонирует с голубым, синим, фиолетовым, белым и черным цветами.

Розовый сочетается с коричневым, белым, оливковым, серым, бирюзовым и нежно-голубым.

Салатовый цвет отлично сочетается с коричневым, серым, темно-синим, а также с серым.

Фуксия (темно-розовый) - с серым, желто-коричневым и просто коричневым.

Черный цвет универсален. Он смотрится превосходно со всеми цветами. Лучше всего с розовым, салатным, белым, красным, желтым или оранжевым

<RMQConfig>

Где:

RMQServerURIList - список серверов RabbitMQ.

ServerURI - URI подключения к серверу.

 

ExchangeList - список точек обмена

Exchange - описание точки обмена. Состоит из имени, под которым т. обмена описана на сервере, и алиаса - уникальное имя в рамках приложений РСДУ. Например, signal обозначает т. обмена сигнальной системы.

 

NamedQueueList- список именованных очередей. (пока не используются)

NamedQueue - описание именованной очереди. Состоит из имени, под которым очередь описана на сервере, алиаса, алиаса т. обмена, к которой очередь привязана и RoutingKey - ключ.

1.     OIC  Протокол используется при высокоскоростном внутрикластерном обмене между модулями технологических серверов. Также может использоваться для передачи массивов данных от вынесенного на выделенные аппаратные средства сервера сбора и передачи (Коммуникационный сервер).

2.     DCP Протокол используется для доступа к данным и для управления данными. Используется при взаимодействиях между клиентскими приложениями и технологическими серверами комплекса. Данное взаимодействие осуществляется через слой доступа к данным - Сервер Данных (oicdcpd).

3.     ADCP               Протокол используется для управления свойствами технологических сервисов ОИК. Как правило, каждый сервис имеет уникальный набор услуг, не ограниченный получением данных и управлением данными. Доступ к таким услугам  осуществляется по ADCP.

Например, ввод/вывод устройства, направления из работы, получение информации о выполняющихся приложениях в кластере, перезагрузка комплекса и т.п.

1.   DPL - Direct Path Load Interface. Интерфейс высокоскоростной загрузки данных в таблицы СУБД Oracle. Является разработкой Oracle, частью прикладного интерфейса OCI.

elregd   Сервер параметров электрического режима   Осуществляет контроль электрических аналоговых параметров. Многоуровневый контроль уставок, сигнализация при выходе за них, дорасчет значений по математическим и пользовательским формулам, замещение значений, расчет усредненных и интегральных характеристик, ведение архивов различных типов.

phregd  Сервер прочих параметров режима    Осуществляет те же функции, что и elreg, для неэлектрических параметров режима.

pwswtd Сервер обработки положений силовых аппаратов      Осуществляет контроль состояний силовых аппаратов, сигнализацию при изменении их положений, обработку команд телеуправления и передачи их серверу сбора и передачи данных. Осуществляет ведение архивов

automatd          Сервер обработки сигналов релейной защиты и автоматики   Осуществляет контроль сигналов релейной защиты и автоматики, сигнализацию при изменении их состояний. Не поддерживается запись архивов.

hgd      Сервер суточных графиков     Осуществляет генерацию значений с использованием математических функций, интерполяции по заданным точкам. Имеет возможность задания изменения параметра на конкретную дату или несколько суток вперед.

arcwrited           Сервер записи архивов            Принимает от технологических серверов, ведущих архивы, срезы данных, укладывает их в локальную БД хоста, передает срезы серверу прямого доступа DPLOAD для вставки их в СУБД на долговременное хранение.

bridged Мост для взаимодействия комплексов            Поддержка связи с альтернативным (резервным) комплексом. Репликация. Контроль статусов кластера или хоста (основной, резервный).

acsrvd  Сервер доступа к услугам подсистем  Доступ к информации о статусе хоста (основной или резервный)

oicdcpd            Сервер данных            Слой доступа к технологическим серверам для клиентского ПО РСДУ5.

ssbsd   Служба рассылки сигналов и сообщений.       Осуществляет ретрансляцию и архивацию сигналов клиентам комплекса. Технологические серверы при возникновении событий сообщают о них службе рассылки сигналов, она в свою очередь осуществляет рассылку клиентам и архивирует сигналы в системные журналы.

lsmagentd         Агент системного монитора     Выполняет функции управления хостом, а также приложениями РСДУ5, установленными на хосте.

dcsd     Сервер сбора и передачи        Выполняет сбор, передачу, ретрансляцию данных, исполнение команд управления (в том числе телеуправление, телерегулирование, синхронизация времени, внеочередной опрос и т.д.) в различных стандартных протоколах. Имеет средства для достоверизации данных по различным критериям, записи передаваемых /собираемых данных в БД.

dpload  Сервер прямого доступа         Принимает от серверов записи архивов комплекса срезы данных, осуществляет вставку в таблицы СУБД Архивов, с использованием интерфейса прямого доступа к данным DPL.

rdarchd Сервер оперативного доступа к архивам        Осуществляет доступ к архивным данным, хранящимся в СУБД

datasrvd           Сервер оперативного доступа к данным          Сервер хранит значения параметров для оперативного доступа к ним. Если при запросе клиента, параметры не находятся в оперативной памяти, то они запрашиваются из БД через Сервер оперативного доступа к архивам и сохраняются для будующего использования. Глубина хранения оперативных данных в сервере задаются индивидуально для каждого типа архивов.

sbridged           Мост для передачи сообщений в другие сети Адресная пересылка сообщений пользователям комплекса, не включенным в технологическую сеть РСДУ5.
