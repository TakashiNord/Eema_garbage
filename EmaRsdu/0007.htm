1 Общие сведения

Данный документ описывает установку Oracle Instant Client, Oracle Instant Client SDK, а также установку ODBC драйвера для Oracle Instant Client в дистрибутивах SLES 10/11/12 и Debian GNU/Linux 7.x-10.x.

Представленная инструкция рассчитана на установку Oracle Instant Client на хост, на котором предполагается установка единственного экземпляра Oracle Instant Client.

Установка нескольких версий клиента Oracle на одном хосте – более сложная задача, которая требует от администратора понимания работы Linux и процесса сборки ПО из исходных кодов.

1.1 Необходимые условия

Необходимы дистрибутивы устанавливаемых компонентов Oracle, скаченные с официальных ресурсов:

ü  instantclient_basic;

ü  instantclient_devel или instantclient_sdk;

ü  instantclient_odbc;

ü  Instantclient_sqlplus.

Операции по установке программных компонентов Oracle требуется выполнять с правами системного пользователя root.

1.2 Необходимые знания

Для выполнения действий, описанных ниже в руководстве, требуются следующие знания:

ü  пароль системного пользователя root.

1.3 Дополнительная информация

Для дистрибутива SLES представлена информация об установке компонентов Oracle из программных пакетов .rpm.

Для Debian описывается способ установки компонентов Oracle из стандартного архива, который может быть применён в любых других дистрибутивах Linux, например, Red Hat или SLES.

Общая информация об устанавливаемых пакетах:

1.    instantclient-basic – является основным пакетом для функционирования клиента Oracle, без которого другие пакеты работать не будут.

Зависит от библиотеки libaio (то есть должен быть установлен пакет libaio).

2.    instantclient-devel – устанавливает заголовочные файлы для сборки ПО РВ РСДУ. Если ПО собирается без ошибок, значит пакет установлен корректно.

instantclient-devel можно не устанавливать, если не предполагается сборка ПО (то есть если ПО на данном сервере не собирается, а устанавливается из пакетов).

Зависит от instantclient-basic (без этого пакета не установится).

3.    instantclient-sqlplus – не обязателен, устанавливает sqlplus. Можно не устанавливать, если sqlplus не требуется. Однако для проверки правильности настроек подключения к БД и разрешения в будущем возможных конфликтов подключения данный пакет настоятельно рекомендуется к установке.

Зависит от instantclient-basic.

4.    instantclient-odbc – не обязателен. Устанавливается, если требуется доступ по ODBC (для какого-либо сервера РСДУ).

Зависит от instantclient-basic.

5.    instantclient-jdbc – не обязателен. Устанавливается, если требуется доступ по jdbc.

Зависит от instantclient-basic.

Таким образом:

ü  подключение через sqlplus проверяет корректность установки instantclient-sqlplus и автоматически instantclient-basic;

ü  сборка ПО РСДУ проверяет корректность установки instantclient-devel и автоматически instantclient-basic;

ü  подключение через odbc проверяет корректность установки instantclient-odbc и автоматически instantclient-basic.

 

2 Установка Oracle Instant Client

2.1 Установка Oracle Instant Client для SLES 10/11/12

1.    Скачать с сайта Oracle (или получить иным способом) rpm-пакет instantclient_basic.

2.    Поместить этот пакет на хост (например, в /root/dist/).

3.    Зайти на хост по ssh.

Следующие действия выполняются на хосте от имени root.

4.    Установить oracle instantclient basic.

Пример:

# zypper --non-interactive install /root/dist/oracle-instantclient11.2-basic-11.2.0.1.0-1.x86_64.rpm

Примечание. Путь установки для x86 и x86-64 различается.

Для x86 – /usr/lib/oracle/11.2/client.

Для x86-64 – /usr/lib/oracle/11.2/client64.

Далее в примерах используется x86-64.

Если установка пакета указанной командой не проходит с выдачей ошибки Package is not signed! и Signature verification failed, применить эту же команду, но без опции --non-interactive; в интерактивном режиме установки ответить на пару вопросов: y (yes) по умолчанию, i (ignore) вместо a (abort), и результат будет положительным.

Пример:

5.    Добавить путь к библиотекам Oracle в пути поиска системы. Для этого создать файл /etc/ld.so.conf.d/oracle.conf с содержимым:

# Oracle

/usr/lib/oracle/11.2/client64/lib

Пример:

# echo "/usr/lib/oracle/11.2/client64/lib" > /etc/ld.so.conf.d/oracle.conf

6.    Обновить список библиотек системы с помощью команды ldconfig.

Пример:

# ldconfig

7.    Создать файл tnsnames.ora в директории /usr/lib/oracle/11.2/client64 с описанием соединения с БД.

Пример:

RSDU =

(DESCRIPTION =

  (ADDRESS =

    (PROTOCOL = TCP)

    (HOST = 192.168.120.1)

    (PORT = 1521)

  )

  (CONNECT_DATA =

    (SERVER = DEDICATED)

    (SERVICE_NAME = RSDU)

  )

)

8.    Создать файл sqlnet.ora в директории /usr/lib/oracle/11.2/client64 с настройками клиента Oracle.

Пример:

NAMES.DIRECTORY_PATH= (TNSNAMES, EZCONNECT)

ADR_BASE = /usr/lib/oracle/11.2/client64

DIAG_SIGHANDLER_ENABLED = FALSE

9.    Для того чтобы приложения могли найти клиента необходимо добавить скрипт, который устанавливает переменные окружения. Для этого нужно создать файл /etc/profile.d/oracle.sh со следующим содержимым:

ORACLE_BASE=/usr/lib/oracle

ORACLE_HOME=$ORACLE_BASE/11.2/client64

ORACLE_SID=RSDU

 

export ORACLE_BASE ORACLE_HOME ORACLE_SID                                                                                                                                                                                                   

 

NLS_LANG=RUSSIAN_CIS.CL8MSWIN1251; export NLS_LANG                                                                                                                                                                                          

NLS_NUMERIC_CHARACTERS=".,"; export NLS_NUMERIC_CHARACTERS                                                                                                                                                                                  

NLS_DATE_FORMAT=='dd.mm.yyyy hh24:mi:ss'; export NLS_DATE_FORMAT                                                                                                                                                                            

 

AGENT_HOME=$ORACLE_HOME/agent                                                                                                                                                                                                  PATH=$PATH:$ORACLE_HOME/bin                                                                                                                                                                                                                 

LD_LIBRARY_PATH=${LD_LIBRARY_PATH:+$LD_LIBRARY_PATH:}$ORACLE_HOME/lib                                                                                                                                                 

 

TNS_ADMIN=$ORACLE_HOME

export AGENT_HOME PATH LD_LIBRARY_PATH TNS_ADMIN

2.2 Установка Oracle Instant Client для Debian GNU/Linux 7.x‑10.x

Перед установкой Oracle Instant client необходимо убедиться, что установлена библиотека libaio1.

# aptitude search libaio

i A libaio1                           - Linux kernel AIO access library - shared lib

Статус i – библиотека установлена, p – отсутствует.

Если команда aptitude search ничего не выведет – библиотека не установлена и в подключенных репозиториях ее тоже нет. В этом случае нужно подключить дополнительные репозитории (DVD-диски дистрибутива или онлайн-репозиторий) и установить библиотеку.

#  sudo apt-get install libaio1

Установка Oracle Instant client:

1.    Скачать с сайта Oracle (или получить иным способом) архив instantclient_basic.

2.    Поместить этот архив на хост (например, в /root/dist/).

3.    Зайти на хост по ssh.

Следующие действия выполняются на хосте от имени root.

4.    Создать корневую директорию для Oracle.

 

Пример:

3 Установка Oracle Instant Client SQLPlus

3.1 Установка Oracle Instant Client SQLPlus для SLES 10/11/12

Перед установкой пакета с sqlplus убедитесь в том, что установлен Oracle Instant client.

1.    Скачать с сайта Oracle (или получить иным способом) rpm-пакет instantclient_sqlplus.

2.    Поместить этот пакет на хост (например, в /root/dist/).

3.    Зайти на хост по ssh.

Следующие действия выполняются на хосте от имени root.

4.    Установить oracle instantclient basic.

Пример:

# zypper --non-interactive install /root/dist/oracle-instantclient11.2-sqlplus-11.2.0.4.0-1.x86_64.rpm

Примечание. Если установка пакета указанной командой не проходит с выдачей ошибки Package is not signed! и Signature verification failed, то примените эту же команду без опции --non-interactive.

5.    Проверка правильности установки и настройки пакетов instantclient_sqlplus и instantclient_basic и файлов окружения ($ORACLE_HOME/tnsnames.ora и /etc/profile.d/oracle.sh) выполняется запуском программы sqlplus с указанием строки логина к БД. Успешное подключение говорит о правильности установки и настройки среды.

Пример:

# sqlplus admin/passme@rsdu

 

SQL*Plus: Release 11.2.0.4.0 Production on Wed Apr 10 11:44 2019

 

Copyright (c) 1982, 2013, Oracle.  All rights reserved.

 

 

Connected to:

Oracle Database 11g Express Edition Release 11.2.0.2.0 - 64bit Production

 

SQL>

Выход из программы sqlplus выполняется по команде exit.

Получение на этапе проверки ошибки ORA-12514 может говорить о некорректном описании БД в файле $ORACLE_HOME/tnsnames.ora (особенное внимание надо уделить правильности строки SERVICE_NAME).

Ошибка ORA-12705 будет говорить о неправильно установленной переменной среды NLS_LANG в файле /etc/profile.d/oracle.sh. Если БД подключения имеет версию XE - Express Edition, следует заменить значение переменной NLS_LANG в файле /etc/profile.d/oracle.sh на AMERICAN_AMERICA.CL8MSWIN1251.

5 Установка ODBC драйвера для Oracle Instant Client

5.1 Установка ODBC драйвера для Oracle Instant Client в SLES 10/11/12

1.    Скачать с сайта Oracle (или получить иным способом) rpm-пакет instantclient_odbc.

2.    Поместить этот пакет на хост (например, в /root/dist/).

3.    Зайти на хост по ssh.

Следующие действия выполняются на хосте от имени root при наличии подключенного к системе инсталяционного диска дистрибутива SLES.

4.    Установить пакет UnixODBC:

# zypper --non-interactive install unixodbc

5.    Для SLES 12 создать символическую ссылку /usr/lib64/libodbc.so.1 на файл /usr/lib64/libodbc.so.2.

# ln -s /usr/lib64/libodbc.so.2 /usr/lib64/libodbc.so.1

6.    Установить Oracle instantclient odbc:

# zypper --non-interactive install /root/dist/oracle-instantclient11.2-odbc-11.2.0.4.0-1.x86_64.rpm

7.    Создать в директории /etc символические ссылки к файлам настройки ODBC (odbcinst.ini и odbc.ini). В SLES 12 эти файлы располагаются в директории /etc/unixODBC/.

Пример:

# ln -s /etc/unixODBC/odbcinst.ini /etc/odbcinst.ini

# ln -s /etc/unixODBC/odbc.ini /etc/odbc.ini

8.    Добавить описание драйвера Oracle ODBC и источник, через который требуется осуществлять подключение.

Для этого можно воспользоваться скриптом odbc_update_ini.sh, который идёт в комплекте с instant_client_odbc и может быть расположен, например, в директории /usr/share/oracle/11.2/client64/.

 

odbc_update_ini.sh <ODBCDM_Home> [<driver_location>] [<driver_name>] [<DSN_name>]

где:     

ODBCDM_Home – корневая директория для поиска конфигурационных файлов ODBC;

driver_location – путь к директории с драйвером Oracle ODBC (в директории должна располагаться библиотека libsqora.so);

driver_name – ODBC имя драйвера в системе;

DSN_name – имя подключения, которое будет создано с помощью указанного драйвера.

Пример:

$ /usr/share/oracle/11.2/client64/odbc_update_ini.sh / /usr/lib/oracle/11.2/client64/lib OracleODBC OracleTopoDB

Примечание. Выполнять команду надо от имени того пользователя, от которого требуется иметь доступ к БД. Это необходимо для того, чтобы скрипт прописал указанное имя соединения нужному пользователю.

9.    Отредактировать файл ~/.odbc.ini в домашней директории пользователя, от имени которого будет запускаться сервер, и прописать в нём верные имя сервиса (ServerName, имя источника, которое указано в файле tnsnames.ora), идентификатор пользователя (UserID) и пароль (Password).

В случае если в файле отсутствуют ServerName, UserID или Password, то их нужно добавить в файл.

Пример:

[OracleTopoDB]

Application Attributes = T

Attributes = W

BatchAutocommitMode = IfAllSuccessful

BindAsFLOAT = F

CloseCursor = F

DisableDPM = F

DisableMTS = T

Driver = OracleODBC

DSN = TopoDB

EXECSchemaOpt =

EXECSyntax = T

Failover = T

FailoverDelay = 10

FailoverRetryCount = 10

FetchBufferSize = 64000

ForceWCHAR = F

Lobs = T

Longs = T

MaxLargeData = 0

MetadataIdDefault = F

QueryTimeout = T

ResultSets = T

ServerName = <Server_Name>

SQLGetData extensions = F

Translation DLL =

Translation Option = 0

DisableRULEHint = T

UserID = <DB_User_Name>

StatementCache=F

CacheBufferSize=20

UseOCIDescribeAny=F

MaxTokenSize=8192

Password = <Password>

10. Создать символическую ссылку /usr/lib64/libodbcinst.so.1, указывающую на /usr/lib64/libodbcinst.so.2 (или /usr/lib/libodbcinst.so.1, указывающую на /usr/lib/libodbcinst.so.2 для x86 версии).

Пример:

# ln -s /usr/lib64/libodbcinst.so.2 /usr/lib64/libodbcinst.so.1

11. Проверить подключение можно командой isql.

Пример:

# isql OracleTopoDB -v
