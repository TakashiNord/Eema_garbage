1  Установка и запуск NGINX для ОС SLES 12

1.1 Общая информация

Документ содержит руководство администратора по установке, настройке и запуску сервера NGINX на SLES 12.

Для установки ОС можно воспользоваться инструкцией «Установка SUSE Linux Enterprise Server 12 SP1 SP3 x64. Руководство администратора».

1.2 Подготовка

Пакет установки доступен для скачивания на официальном сайте nginx.org для выбранной ОС.

Чтобы получить пакет для SLES 12, необходимо перейти по ссылке: http://nginx.org/packages/sles/12/x86_64/RPMS/.

1.3 Установка

Для установки, необходимо перейти в каталог с .rpm пакетом nginx и выполнить команду:

# rpm -i ./nginx-1.14.0-1.sles12.ngx.x86_64.rpm

1.4 Основные команды для работы с NGINX

ü  systemctl start nginx – запуск;

ü  systemctl stop nginx – остановка;

ü  systemctl status nginx – проверка состояния;

ü  systemctl reload nginx – загрузка новой конфигурации. При этом предварительно проходит проверка конфигурационных файлов и, если ошибок не выявлено, происходит ее применение для новых соединений. Старые соединения продолжают работу с предыдущей версией.

1.5 Расположение файлов NGINX

Директория для размещения web-содержимого: /usr/share/nginx/html.

1.5.1 Конфигурационные файлы

ü  /etc/nginx/ - директория с файлами конфигурации;

ü  /etc/nginx/nginx.conf – основной конфигурационный файл nginx;

ü  /etc/nginx/conf.d/ - директория с дополнительными файлами конфигурации (включаются в конфигурацию директивой include в nginx.conf);

ü  /etc/nginx/conf.d/default.conf – настройки по умолчанию для виртуальных хостов (блоки server.)

1.5.2 Лог файлы NGINX

Запись в лог настраивается в конфигурации NGINX. По умолчанию пишутся 2 лог файла:

ü  /var/log/nginx/access.log – фиксируются запросы к серверу;

ü  /var/log/nginx/error.log – ошибки.

Примечание. Если отсутствует каталог для записи лог-файлов (/var/log/nginx), NGINX не запускается.

 

Приложение А. Установка и подключение модулей NGINX

Модульная архитектура NGINX позволяет расширять его функциональность средствами дополнительных модулей, подключаемых при необходимости.

Дополнительные пакеты, в т.ч. модули NGINX, доступны по ссылке:

http://nginx.org/packages/sles/12/x86_64/RPMS/

В текущей конфигурации NGINX, используемой для работы служб РСДУ, дополнительные модули не используются.

А.1 Модуль njs

Сценарный язык njs - это подмножество языка JavaScript, позволяющее расширить функциональность nginx.

Модуль поддержки njs поставляется в виде отдельного пакета.

Скачать можно по адресу:

http://nginx.org/packages/sles/12/x86_64/RPMS/

Установка:

# rpm -i ./nginx-module-njs-1.14.0.0.2.5-1.sles12.ngx.x86_64.rpm

warning: ./nginx-module-njs-1.14.0.0.2.5-1.sles12.ngx.x86_64.rpm: Header V4 RSA/SHA1 Signature, key ID 7bd9bf62: NOKEY

--------------------------------------------------------------

The njs dynamic modules for nginx have been installed.

To enable these modules, add the following to /etc/nginx/nginx.conf

and reload nginx:

 

    load_module modules/ngx_http_js_module.so;

    load_module modules/ngx_stream_js_module.so;

 

Please refer to the modules documentation for further details:

http://nginx.org/en/docs/njs/

http://nginx.org/en/docs/http/ngx_http_js_module.html

http://nginx.org/en/docs/stream/ngx_stream_js_module.html

Для подключения к NGINX необходимо в файле конфигурации nginx.conf до объявления директив events и http прописать строки, упомянутые в выводе после завершения установки модуля:

load_module modules/ngx_http_js_module.so;

load_module modules/ngx_stream_js_module.so;

Загрузить новую конфигурацию:

# systemctl reload nginx
